# Warp 코드 리뷰 패널 마스터하기

[영상 바로가기](https://youtu.be/4JlN0rvoZA8)

## 핵심 요약

- Warp의 코드 리뷰 패널을 사용하면 코딩 중 몰입 상태를 유지하면서 변경 사항을 효율적으로 관리할 수 있음
- 패널에서 수정된 파일, 추가/삭제된 코드 라인, 모든 diff를 한눈에 확인 가능
- Warp 내장 에디터로 파일을 직접 열어 편집하고 변경 사항을 즉시 확인
- 기존 diff를 컨텍스트로 추가하여 일관된 코드 변경을 AI에게 요청 가능
- 패널에서 직접 커밋과 push를 수행하여 워크플로우 간소화
- 현재 브랜치와 main 브랜치 간 diff를 빠르게 비교 가능

## 전체 번역

안녕하세요, 저는 Jess입니다. Warp의 개발자 애드보킷(developer advocate)이고, 오늘은 코드 리뷰 패널을 마스터하는 방법을 알려드려서 여러분이 코딩하면서 더 잘 몰입 상태를 유지할 수 있도록 돕겠습니다. 저는 AI Observe라는 리포지토리에서 작업하고 있는데, 이것은 AI 모델에서 토큰이 어떻게 계산되는지 시각화하기 위해 제가 만든 작은 프로젝트입니다. 이것은 여러분의 텍스트 입력이 토큰으로 어떻게 분해되는지, 요청에서 얼마나 많이 사용되는지 보여주고, 개발자들이 그것이 비용과 모델 사용 측면에서 무엇을 의미하는지 이해할 수 있도록 도와줍니다. 저는 주말에 약 1시간 만에 이것을 급하게 만들었습니다. 그래서 보시다시피 웹 앱이 아직 꽤 거칩니다. 예를 들어 여기 호버 텍스트가 어두운 회색 배경에 검은색이어서 읽기 어렵습니다. Warp를 사용해서 그것을 어떻게 고치는지 살펴보겠습니다. 저는 Warp에게 직접 물어보는 것으로 시작하겠습니다. "배경에 대비해서 읽기 쉽도록 이 호버 텍스트를 업데이트해줄 수 있나요?" 더 명확하게 하기 위해 이미지도 첨부하겠습니다. 그런 다음 에이전트 프로세스를 시작하면 Warp가 제안된 모든 변경 사항이 포함된 코드 diff를 생성하기 시작합니다. 이제 이것은 훌륭하지만, 더 큰 변경 사항의 경우 수정된 모든 파일을 보기 어려울 수 있고, 그것들을 찾기 위해 UI를 계속 위로 스크롤해야 하는 것을 원하지 않습니다. 그래서 새로운 코드 리뷰 패널이 등장합니다. 왼쪽 상단의 view changes 버튼이나 입력창의 get dirty 칩을 클릭해서 코드 리뷰 패널을 열 수 있습니다. 이것들은 리포지토리 내에서만 작동한다는 점을 알아두세요.

코드 리뷰 패널에서 열려 있는 변경 사항에 얼마나 많은 파일이 있는지, 얼마나 많은 코드 라인이 추가되거나 삭제되었는지, 만들어진 모든 다른 코드 diff를 볼 수 있고, 이 패널을 열면 수정된 라인 수와 함께 수정된 모든 파일의 더 구조화된 분석을 제공합니다. 이제 더 깊이 들어가고 싶다면, Warp의 내장 에디터에서 직접 파일을 열 수 있습니다. 구문 강조, 찾기 및 바꾸기, 기본 편집 도구가 있는 경량 에디터입니다. 예를 들어, 여기서 색상 스키마를 수정하고 저장하겠습니다. 돌아가면 변경 사항이 이미 코드 diff 패널에 반영되어 있습니다. 그리고 제 앱에서도 그 변경 사항이 반영된 것을 확인할 수 있습니다. 이제 이 호버 스타일이 한 파일만이 아니라 앱 전체에 일관되게 적용되기를 원한다고 가정해봅시다. Warp에게 호버 효과를 컴포넌트화해서 새로운 색상 스키마가 호버 UI가 나타나는 모든 곳에 적용되도록 프롬프트할 수 있습니다. 그리고 방금 만든 diff를 프롬프트의 컨텍스트로 추가할 수 있습니다. 그러면 이렇게 diff를 인용하는 것을 볼 수 있습니다. 이제 에이전트가 완료되면 파일 트리로 들어가서 생성된 새 컴포넌트를 보고 클릭해서 빠르게 확인할 수 있습니다. 에이전트는 그것을 tool tip 컴포넌트라고 이름 지었습니다. 마우스 이벤트를 올바르게 처리하고, 제가 만든 UI와 색상 스키마를 유지하는 것 같습니다.

저에게는 좋아 보입니다. 그리고 변경 사항에 만족하면 패널에서 직접 커밋하고 push할 수 있습니다.

그러면 패널이 제로 상태로 재설정되어 열려 있는 변경 사항이 없다는 것을 알려줍니다. 그리고 여기를 클릭하면 현재 브랜치와 main 사이의 diff를 빠르게 볼 수 있습니다. Warp 코드 리뷰 패널은 변경 사항을 검토하고, 편집하고, 커밋하는 것을 더 빠르고 직관적으로 만듭니다. 저는 코딩하는 동안 이 패널을 계속 열어두는 것을 좋아하는데, 도구를 전환하거나 집중을 잃지 않고도 제 리포지토리에서 일어나는 모든 것을 실시간으로 개괄적으로 볼 수 있기 때문입니다.

한번 시도해보시고 그것이 여러분의 코딩 방식을 어떻게 바꾸는지 확인해보세요.
