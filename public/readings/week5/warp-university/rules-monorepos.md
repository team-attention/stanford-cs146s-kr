# 규칙으로 모노레포 연결하기

[영상 바로가기](https://youtu.be/bndY6opaA7w)

## 핵심 요약

- **모노레포 연결**: 여러 연관된 저장소를 규칙으로 연결하여 에이전트가 프로젝트 간 관계를 이해하도록 설정
- **Warp 사례**: 서버(Warp Server), 클라이언트(Warp Internal), API 스키마(Warp Proto APIs) 세 저장소를 규칙으로 연결
- **자동화 효과**: 에이전트가 스키마를 업데이트하면 자동으로 관련 저장소로 이동하여 타입을 업데이트하는 워크플로우 실현
- **전역 규칙 권장**: 프로젝트 간 관계는 에이전트의 전역 규칙으로 설정하는 것이 효과적

---

## 전체 번역

서로 연관된 프로젝트가 있고 그것들이 어디에 있는지 설명하세요. Warp에서는 세 개의 저장소가 서로 연관되어 있습니다. 서버 측 코드가 있는 Warp Server, 클라이언트 앱인 Warp Internal, 그리고 이들이 서로 통신할 때 사용하는 API 스키마를 저장하는 Warp Proto APIs가 있습니다. 이는 에이전트의 전역 규칙으로 설정하는 것이 가장 좋으며, 저는 Warp에서 이렇게 설정했습니다. 여기에 우리가 가진 세 개의 저장소와 저장소 간 스키마 업데이트 및 타입 업데이트 관행을 설명했습니다.

이것은 앞으로 변경 사항을 만들 때 매우 유용합니다. 코딩 에이전트가 스키마를 업데이트하자마자, warp server로 CD하여 거기서 타입을 업데이트해야 한다는 것을 알게 됩니다. 그런 다음 클라이언트에서도 동일하게 수행합니다.
