# Warp 에이전트 런타임 첫 만남

[영상 바로가기](https://www.youtube.com/watch?v=hV6UdEf3C1I)

## 핵심 요약

- **Warp CLI 소프트 런칭**: Warp의 에이전트를 어떤 환경에서도 사용할 수 있게 해주는 CLI 도구
- **세 가지 활용 사례**:
  1. **Cursor 터미널**: 코드베이스 요약, 다이어그램 생성, 아키텍처 분석
  2. **GitHub Actions**: PR 자동 리뷰, API 키 노출 검사, 에러 핸들링 검증
  3. **원격 서버**: Docker 로그 요약, 메모리/DB/API 이슈 파악
- **핵심 기능**:
  - 저장된 프롬프트, Warp Drive 컨텍스트, 에이전트 프로필 재사용
  - YOLO 에이전트 같은 자율 프로필 활용
  - CI/CD 환경에서 헤드리스 실행
- **현재 상태**: 베타 단계, 소규모 테스터 모집 중

## 전체 번역

안녕하세요 여러분. Warp CLI의 소프트 런칭을 발표하게 되어 기쁩니다. Warp CLI는 Warp의 에이전트를 어떤 환경에서도 사용할 수 있게 해줍니다.

Warp 내부에서 사용하는 저장된 프롬프트, Warp Drive 객체, 에이전트 프로필, MCP 서버를 어디서나 적용할 수 있습니다. 세 가지 활용 예시를 보여드리겠습니다.

여기 Cursor의 터미널에 있습니다. Warp의 에이전트를 사용해서 제 코드베이스를 요약하고 싶습니다. Warp CLI를 설정하면 이제 Warp CLI를 실행해서 다이어그램과 함께 전체 요약을 받을 수 있습니다. 컴포넌트 간 상호작용, 배포 아키텍처, 런타임 동작 같은 코드베이스 개요도 제공됩니다. 여기서 핵심 가치는 Warp 내부에 있지 않더라도 여전히 저장된 모든 프롬프트, Drive 컨텍스트, 에이전트 프로필에 접근할 수 있다는 점입니다. 예를 들어, 저는 이미 Warp에 YOLO 에이전트라는 에이전트 프로필을 설정해두었습니다. 이 프로필은 자율성에 최적화되어 있습니다. 항상 명령을 실행하고, 항상 파일을 읽으며, 안전을 위한 명령 거부 목록이 있습니다.

이 프로필의 ID는 warp agent profile list를 실행하면 알 수 있습니다. 이 명령은 제 모든 프로필과 ID를 보여줍니다. 또한 코드베이스를 요약해주는 저장된 프롬프트도 이미 가지고 있었습니다. 배포 아키텍처를 설명하라는 지시사항이 설정되어 있습니다. 그리고 첨부된 규칙 덕분에 제가 요약을 요청할 때마다 자동으로 다이어그램을 생성합니다. 이 프롬프트의 ID는 Warp 내부의 @ 메뉴를 사용해서 찾을 수 있습니다. 두 번째 예시는 GitHub Actions에서 Warp CLI를 실행하는 것입니다. 여기서는 github/workflows 아래에 YAML 파일을 설정했습니다.

이 워크플로우는 Warp CLI를 설치하고, 인증하고, PR에 대한 패치 파일을 생성한 다음, 저장된 프롬프트와 프로필로 리뷰어 에이전트를 실행합니다.

예를 들어, 노출된 API 키, 누락된 에러 핸들링에 대해 경고하고, 더 나은 로깅 같은 긍정적인 개선사항도 강조합니다. 이 모든 것이 PR에 새로운 커밋을 푸시할 때마다 자동으로 발생합니다. 가치는 제가 Warp에서 로컬로 신뢰하는 동일한 리뷰어 워크플로우를 재사용하는데, 이제 동일한 컨텍스트로 지속적 통합 환경에서 헤드리스로 실행된다는 것입니다. 세 번째 예시는 원격 머신에서 Warp CLI를 실행해서 서버 로그를 요약하는 것입니다. Docker 컨테이너가 서버로 로그를 스트리밍하고 있다고 상상해보세요. Warp CLI를 실행하면 메모리 고갈, 데이터베이스 연결 실패, 외부 API 타임아웃, 높은 디스크 사용량, 인증 오류 등 무슨 일이 일어나고 있는지 명확한 요약을 받을 수 있습니다. 끝없는 로그 라인을 살펴보는 대신, 한 곳에서 중요한 이슈들을 얻을 수 있습니다. Cursor에 있든, GitHub Actions를 통해 PR을 리뷰하든, 원격 서버에서 로그를 검토하든, Warp CLI는 제 에이전트에 대한 일관된 접근을 제공합니다. 이것이 베타 런칭이라는 점을 빠르게 알려드립니다. 지금은 소규모 테스터 그룹을 모으는 데 집중하고 있으며, 목표는 무엇이 잘 작동하고 무엇이 부족한지에 대한 여러분의 피드백을 받는 것입니다. 여러분 모두가 이것에 대해 흥분하기를 바라며, Warp CLI로 무엇을 만들어낼지 기대됩니다.
