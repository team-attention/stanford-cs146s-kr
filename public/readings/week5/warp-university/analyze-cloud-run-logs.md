# Cloud Run 로그 분석하기

[영상 바로가기](https://youtu.be/GJ0NepZmmv8)

## 핵심 요약

- Warp는 코딩뿐만 아니라 클라우드 서버 관리, Docker/Kubernetes 작업 등 다양한 개발 태스크를 수행할 수 있는 도구입니다
- 음성 입력 기능을 지원하여 자연어로 복잡한 작업을 요청할 수 있습니다
- Cloud Run 인스턴스에서 로그를 가져와 info/warning/error 레벨로 분류하고, 히스토그램과 리포트를 자동 생성합니다
- 에이전트 모드에서 자동으로 필요한 명령어와 Python 스크립트를 작성하고 실행합니다
- 실행 과정을 실시간으로 확인할 수 있으며, 언제든지 중단하거나 빨리 감기할 수 있습니다
- 복잡한 클라우드 로그 분석 작업을 자연어 프롬프트 하나로 완료할 수 있는 생산성 도구입니다

## 전체 번역

안녕하세요, 오늘은 Warp를 사용하여 백엔드 및 프로덕션 작업을 수행하는 방법을 보여드리겠습니다. Warp의 가장 멋진 점 중 하나는 코딩부터 클라우드 서버 작업, Docker와 Kubernetes 작업까지 모든 종류의 개발 태스크를 수행하도록 프롬프트를 줄 수 있다는 것입니다. 오늘은 Warp에게 우리 클라우드 서버 중 하나에서 로그를 가져와서 약간의 분석을 하도록 해보겠습니다. 이전 데모에서처럼, 저는 Warp에게 음성으로 말할 것입니다. 이 음성 입력 버튼을 클릭하면 항상 음성 모드를 활성화할 수 있고, 단축키를 설정할 수도 있습니다.

"warp-server-staging g-cloud 프로젝트를 사용해서 warp-server Cloud Run 인스턴스에서 지난 10분간의 로그를 가져와줘. 그 로그를 info, warning, error 레벨로 정리해주고, 그 로그 메시지 유형별로 히스토그램과 리포트를 만들어줘. 특히 내가 조사해야 할 가장 우려되는 에러들이 무엇인지 구체적으로 알려줘."
