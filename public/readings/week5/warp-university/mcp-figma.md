# Figma MCP 서버 사용하기

[영상 바로가기](https://youtu.be/C0g_Onjtsb8)

## 핵심 요약

- **Figma MCP 서버**: Figma 디자인 데이터를 가져와 코드로 자동 변환
- **설정 방법**: Figma에서 개인 액세스 토큰 생성 → Warp MCP 패널에 설정 추가
- **사용 방법**: Figma에서 디자인 선택 → 링크 복사 → Warp 프롬프트로 실행
- **생성 결과**: HTML/CSS 코드가 자동 생성되며, 레이아웃·폰트·색상을 재현
- **장점**: 보일러플레이트 작성 시간 절약, 빠른 프로토타이핑
- **한계**: 그라디언트 등 일부 세부 스타일은 누락될 수 있음

## 전체 번역

Figma MCP 서버를 사용하면 Warp 에이전트가 Figma에서 디자인 데이터를 가져와서 코드 스캐폴딩으로 변환할 수 있습니다. 설정 방법은 다음과 같습니다. Figma 계정에 들어가서 프로필 사진을 클릭하고, 설정(Settings)으로 이동한 다음, 보안(Security) 섹션으로 가서 새로운 개인 액세스 토큰을 생성합니다. Warp에서 전체 읽기 권한을 부여합니다. MCP 패널을 열고, Figma JSON 설정을 붙여넣고, 방금 받은 액세스 토큰을 입력한 다음 저장을 누릅니다. 이 데모에서는 Warp의 재디자인된 홈페이지 디자인을 사용하겠습니다. Figma에서 원하는 그룹을 선택하고, 오른쪽 클릭한 다음, "선택 항목에 대한 링크 복사(Copy link to selection)"를 선택합니다. 이제 Warp로 돌아가서, Warp Drive에 저장해둔 "Figma mockup"이라는 프롬프트를 사용합니다. 이 프롬프트는 에이전트에게 디자인을 코드로 재현하도록 지시하며, 원본 섹션, 간격, 폰트 크기, 색상 및 일반적인 모범 사례를 유지하도록 명시합니다. 실행하면 MCP 서버가 Figma 데이터를 가져오고(get Figma data) Figma 이미지를 다운로드(download Figma images)합니다.

그런 다음 전체 레이아웃에 대한 HTML과 CSS를 생성하기 시작합니다. 화면에서 디자인 데이터를 가져오고, 코드 파일을 하나씩 생성한 다음, 최종 결과를 브라우저에서 미리보는 과정을 볼 수 있습니다. 디자인과 생성된 파일을 번갈아 보면 놀라울 정도로 비슷합니다. 헤더, 기능 카드, 코드 생성 섹션이 모두 포함되어 있습니다. 그라디언트나 푸터와 같은 일부 세부사항은 누락되어 있지만, 이것은 훌륭한 출발점입니다. 저는 이 기능이 보일러플레이트를 건너뛰고 더 빠르게 반복 작업을 진행하려는 PM, 디자이너 또는 개발자에게 엄청난 시간 절약이 된다고 생각합니다.
