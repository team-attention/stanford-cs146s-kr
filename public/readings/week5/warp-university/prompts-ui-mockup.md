# 목업과 일치하는 UI 실제로 코딩하기

[영상 바로가기](https://youtu.be/GA0OO_1o8LA)

## 핵심 요약

- **문제**: AI는 구체적인 디자인 비전을 세부적으로 구현할 때 원하는 대로 나오지 않는 경우가 많음
- **해결책**: AI에게 UI 엔지니어 역할을 부여하고 구조화된 마크다운으로 전체 사양을 먼저 작성하도록 요청
- **2단계 프롬프트**: 1) 사양 생성 → 2) 특정 기술 스택(React, Tailwind 등)으로 구현 요청
- **핵심 팁**: 자신이 익숙한 기술 스택을 명시하고, 완성 후 결과를 반드시 검증할 것

---

## 전체 번역

AI로 디자인을 작업할 때 흔히 겪는 문제가 있습니다. 보통 우리는 매우 구체적인 비전을 가지고 있습니다. 그런데 세부적으로 들어가려고 하면, 제대로 나오지 않습니다.

우리처럼 디자인에도 시각적 이해가 필요합니다. 그래서 이를 위해 더 나은 프롬프트를 작성하는 방법이 있습니다. 이 방법은 전체 사양을 구조화된 마크다운 파일로 제공합니다. AI에게 UI 엔지니어라는 정체성을 부여한 점에 주목하세요.

응답을 받으면 다른 프롬프트를 사용할 수 있습니다. 저는 React와 Tailwind를 사용하고 싶습니다.

그러면 최종 구현에 더 가까운 결과를 얻게 됩니다. 중요한 점은 자신이 편한 기술 스택을 포함하고, 완성되면 작업을 확인하는 것입니다. 항상 완벽하지는 않겠지만, 이전보다 훨씬 더 가까워질 것입니다.
