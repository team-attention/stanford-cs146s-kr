# Warp에서 MCP 서버 사용하기

[영상 바로가기](https://www.youtube.com/watch?v=8vn2brhJrF8)

## 핵심 요약

- **MCP 서버 통합**: Warp은 Linear와 같은 외부 서비스와 통합할 수 있는 MCP 서버를 지원합니다
- **동적 컨텍스트**: 대화 중간에 MCP 서버를 추가해도 즉시 다음 메시지부터 컨텍스트로 활용됩니다
- **티켓 자동 해결**: Linear MCP 서버를 통해 AI가 티켓 정보를 자동으로 가져와 문제를 이해하고 해결책을 구현합니다
- **최소한의 사용자 입력**: AI가 필요한 컨텍스트를 직접 수집하고 기존 코드를 참고하여 자동으로 작업을 완료합니다

## 전체 번역

접근 권한을 부여해야 하는 더 중요한 회사 데이터나 여러 출처에서 한 문장만 얻으려고 합니다.

안녕하세요, 저는 Andrew입니다. Warp의 소프트웨어 엔지니어이고 오늘은 MCP 서버 지원에 대해 이야기하겠습니다.

이것은 우리의 기존 입력 형식이고, 우리는 유니버설 입력이라고 부르는 새로운 형식이 있습니다. 기존 모드에서는 여기에 칩이 지원되는데, 이는 여기에 표시된 가상 환경을 활성화했을 때 알려줍니다. 그리고 유니버설로 전환하면 그 칩을 사용할 수 없는 것을 볼 수 있습니다. 이 티켓을 해결하도록 도와주세요.

이 시점에서 Warp은 티켓이 무엇인지, 티켓을 어디서 찾아야 하는지 전혀 모릅니다. 우리는 Linear를 통해 티켓팅을 하는데, Linear에는 우리가 지원을 추가할 수 있는 MCP 서버가 있습니다. 들으시면 알 수 있듯이, GitHub을 확인하려고 하지만 아무것도 할 수 없습니다. 그래서 이것을 취소하고 Linear MCP 서버를 추가하겠습니다.

여기서 추가를 클릭하고 JSON을 붙여넣기만 하면 됩니다. 그러면 여기서 MCP 서버가 시작되고 티켓 읽기와 쓰기에 사용할 수 있는 도구 목록이 있습니다. 우리가 할 수 있는 또 하나의 좋은 것은 여기에 Linear 확인이라고 부를 규칙을 추가하는 것인데, 이를 통해 여기 티켓이 있다고 말하고 Linear에서 티켓을 확인해야 한다고 알려줄 수 있습니다. 이제 새 대화를 다시 시작하고 이 티켓을 해결하도록 도와달라고 말하면

[Music] 우리가 MCP 지원을 구현한 멋진 방법 중 하나는 매우 동적이도록 허용한 것입니다. 따라서 MCP 서버 없이 대화를 시작하고 대화 중간에 서버를 시작하면 다음 메시지에서 그것을 컨텍스트로 사용할 수 있습니다. 이제 MCP 서버를 성공적으로 식별한 것을 볼 수 있습니다. 실행을 눌러보겠습니다.

그리고 여기서 Linear에서 직접 컨텍스트를 가져올 수 있었던 것을 볼 수 있습니다. cargo run을 실행해서 기능이 작동하는지 확인해보겠습니다. 보세요.

꽤 간단합니다. 바로 작동했습니다. Linear로 가서 해결해야 할 문제를 파악하는 데 필요한 모든 컨텍스트를 가져오고, 코드를 살펴보며 다른 칩이 어떻게 구현되었는지 식별하고, 결국 제 입력이 거의 없이 이것을 구축했습니다. 그냥 작동했습니다.
