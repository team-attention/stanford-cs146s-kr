# Few-shot 프롬프팅

## 개요

Few-shot 프롬프팅은 프롬프트 내에 예시를 제공하여 모델 성능을 향상시키는 인컨텍스트 학습(in-context learning) 기법입니다. 제로샷 능력에만 의존하지 않고, 예시를 통해 모델이 더 나은 응답을 생성하도록 유도합니다.

## 핵심 개념

Few-shot 프롬프팅은 프롬프트에 시연을 포함하여 모델 성능을 높이는 인컨텍스트 학습 기법입니다. 시연은 패턴을 확립하여 모델이 더 복잡한 작업을 처리하는 데 도움을 줍니다.

## 역사적 맥락

모델이 충분한 규모에 도달하면서 이 능력이 나타났습니다. 연구에 따르면, 특정 임계점에 도달하기 전까지 작은 모델에서는 few-shot 특성이 나타나지 않았습니다.

## 실제 예시

대표적인 활용 사례로, 모델에게 새로운 단어를 문장에서 사용하도록 가르치는 것이 있습니다.

**작업**: 새로 소개된 단어를 문장에서 올바르게 사용하기 (단어당 하나의 예시 제공)

**결과**: 모델은 단 하나의 시연만 보고도 적절한 문장을 생성했습니다. 최소한의 입력에서 패턴을 내재화했음을 보여주는 결과입니다.

## 연구 기반 설계 원칙

주요 연구 결과에 따르면:

- 레이블 정확도와 무관하게 레이블 공간과 입력 분포가 중요
- 무작위 레이블을 사용해도 형식 일관성이 성능을 향상시킴
- 균등 분포보다 실제 분포에서 레이블을 선택하는 것이 더 효과적

## 주요 한계

Few-shot 프롬프팅은 복잡한 추론 작업에서 어려움을 겪습니다. 산술 추론 문제 테스트에서 여러 시연을 제공해도 잘못된 답을 생성했습니다. 정교한 인지 작업에는 더 발전된 방법이 필요합니다.

## 다음 단계

Few-shot 접근 방식이 충분하지 않을 때는 Chain of Thought 프롬프팅이나 파인튜닝 전략을 검토해 보세요.

- 모델 규모가 특정 임계점을 넘어야 이 능력이 발현됨
