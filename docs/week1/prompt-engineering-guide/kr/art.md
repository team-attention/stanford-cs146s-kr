# 자동 추론 및 도구 사용(ART)

## 개요

ART는 Chain of Thought 프롬프팅과 도구 통합을 교차 방식으로 결합한 프레임워크입니다. 연구에 따르면 이 프레임워크는 "고정된(frozen) LLM으로 중간 추론 단계를 프로그램 형태로 자동 생성"합니다.

## ART의 작동 방식

다음 단계로 구성됩니다:

1. **선택**: 새로운 태스크를 만나면 시스템이 태스크 라이브러리에서 관련 있는 다단계 추론 및 도구 사용 예시를 검색합니다
2. **실행**: 테스트 중 외부 도구 호출이 필요하면 생성을 일시 중지합니다
3. **통합**: 도구 출력을 통합한 후 생성을 재개합니다

이 접근 방식으로 모델은 예시를 바탕으로 일반화하고, 익숙하지 않은 태스크를 분해하며, 적절한 도구를 언제 활용할지 파악합니다. 태스크별 학습이 필요 없습니다.

## 주요 장점

- **확장성**: 사람이 라이브러리를 업데이트하여 추론 단계를 개선하거나 새로운 도구를 추가할 수 있습니다
- **성능**: ART는 이전에 접하지 않은 BigBench와 MMLU 태스크에서 few-shot 프롬프팅과 자동 Chain of Thought를 크게 능가합니다
- **유연성**: 사람의 피드백을 반영하면 수작업 Chain of Thought 프롬프트보다 뛰어난 성능을 냅니다

## 출처

Paranjape 외(2023)가 이 프레임워크를 소개했으며, 언어 모델을 활용한 복잡한 문제 해결에서 추론과 도구 통합을 결합한 중요한 발전입니다.

## 핵심 요약

- ART는 Chain of Thought 프롬프팅과 도구 사용을 결합한 프레임워크
- 고정된(frozen) LLM으로 중간 추론 단계를 자동 생성
- 태스크 라이브러리에서 관련 예시를 검색해 새로운 태스크에 적용
- BigBench, MMLU에서 few-shot 프롬프팅 대비 우수한 성능
- 사람의 피드백으로 확장 가능
