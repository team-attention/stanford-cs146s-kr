# 5. 정직함, 페르소나, 은유

[영상 바로가기 (24:27)](https://www.youtube.com/watch?v=T9aRN5JkmL8&t=1467s)

## 요약

모델에게 역할을 부여하는 것보다 솔직하게 상황을 설명하는 것이 더 효과적인 이유를 설명합니다.

**핵심 포인트:**
- 역할 부여(페르소나)는 예전 모델에서 더 효과적이었음
- 현재 모델에게는 거짓말보다 솔직한 설명이 낫다
- "파견 회사에서 온 유능한 사람" 비유 - 맥락 없지만 능력 있는 사람에게 설명하듯
- pretrained model과 RLHF model에 대한 직관이 다름
- 실제 작업을 직접 설명하는 것이 가장 효과적

## 전체 번역

[24:27] 사실 이 부분을 좀 더 자세히 다루고 싶어요. 이게 가장 유명한 프롬프팅 팁 중 하나거든요. 언어 모델에게 어떤 페르소나나 역할을 부여하는 거죠. 제 경험상 결과가 좀 엇갈리는 것 같아요. 예전 모델에서는 좀 더 효과가 있었는데, 이제는 그렇게까지는 아닌 것 같기도 하고요. Amanda, 당신은 항상 모델에게 상황을 솔직하게 설명하는 걸 봤어요. "저는 AI 연구자고 이런 실험을 하고 있어요"라고 하면서요.

- 저는 제가 누군지 말해요.
- 제 이름도 알려주고, "당신이 대화하고 있는 사람은 이런 사람이에요"라고요.

모델에게 거짓말을 하거나 "팁으로 500달러 드릴게요" 같은 걸 강요하는 대신, 그런 수준의 솔직함이 더 좋다고 생각하세요?

- 네. 모델이 더 유능해지고 세상에 대해 더 많이 이해하게 되면서, 굳이 거짓말을 할 필요가 없다고 봐요. 그리고 저는 원래 거짓말 자체를 싫어해서 모델에게 거짓말하는 것도 싫어요.

[31:05] - 제가 처음으로 시도해보라고 권하는 건, 예전에 사람들에게 이런 사고 실험을 제안했어요. 어떤 작업이 있다고 상상해보세요. 파견 회사에서 이 일을 할 사람을 보내달라고 했어요. 이 사람이 도착했는데, 꽤 유능한 사람이에요. 당신의 업계에 대해서도 많이 알고 있고요. 하지만 회사 이름은 몰라요. 말 그대로 방금 도착해서 "안녕하세요, 여기서 할 일이 있다고 들었는데, 설명해 주세요"라고 하는 거예요. 그러면 "그 사람에게 뭐라고 말할 건가요?"

## 핵심 요약

- 페르소나/역할 부여는 예전 모델에서 더 효과적이었음
- 현재 모델에게는 솔직하게 상황을 설명하는 것이 더 좋음
- "파견 회사 직원" 비유: 유능하지만 맥락이 없는 사람에게 설명하듯이
- 거짓말("팁 500달러")보다 진실된 설명이 효과적
