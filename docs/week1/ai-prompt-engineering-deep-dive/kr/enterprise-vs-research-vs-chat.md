# 7. 엔터프라이즈 vs 연구 vs 일반 채팅 프롬프트

[영상 바로가기 (45:18)](https://www.youtube.com/watch?v=T9aRN5JkmL8&t=2718s)

## 요약

세 가지 유형의 프롬프트 작성 방식 차이를 비교합니다.

**핵심 포인트:**
- 연구용: 다양성과 유연성 추구, 예시를 적게 사용
- 엔터프라이즈: 신뢰성과 일관성 중시, 예시를 많이 사용
- 연구용 예시는 실제 데이터와 의도적으로 다르게 (설명적 예시)
- 일반 채팅: 한 번 성공하면 끝
- 엔터프라이즈: 백만 번 성공해야 함 - 전체 입력 범위 테스트 필요

## 전체 번역

[45:19] 이건 아마 더 일찍 했어야 할 것 같은데, 지금 하겠습니다. 엔터프라이즈 프롬프트와 연구용 프롬프트, 그리고 Claude.ai에서의 일반 채팅 프롬프트의 차이를 명확히 해볼까요. Zack, 당신은 고객 작업부터 연구까지 전 영역을 아우르고 있잖아요. 각각이 무엇을 의미하는지 설명해주실래요?

- 이 방에 계신 분들께 말씀드리자면, 저는 Amanda의 Claude 채널에서 읽는 프롬프트와 David이 작성하는 프롬프트를 비교해서 생각해요. 두 가지 모두 들어가는 세심함과 뉘앙스 측면에서 매우 비슷해요. 연구용의 경우에는 다양성과 변화를 훨씬 더 많이 추구하는 것 같아요.

한 가지로 요약하자면, Amanda는 예시를 많이 넣거나, 하나 두 개 넣는 걸 별로 좋아하지 않는다는 걸 알게 됐어요. 너무 적으면 모델이 그것에 고착되거든요. 반면에 제가 쓰거나 David이 쓰는 프롬프트에서는 예시를 많이 넣어요. 저는 예시를 미친 듯이 추가하는 걸 좋아해요. 쓰러질 것 같을 정도로 많이 넣을 때까지요.

그 이유는 소비자용 애플리케이션에서는 신뢰성을 정말 중요하게 여기기 때문이라고 생각해요. 형식을 엄청나게 신경 쓰고, 모든 답변이 동일해도 괜찮아요. 반면에 연구용으로 프롬프트를 작성할 때는 모델이 탐색할 수 있는 가능성의 범위를 정말로 활용하려고 해요.

- 네. 저는 예시를 줄 때, 실제로 모델이 볼 데이터와 다른 예시를 만들려고 해요. 의도적으로 설명을 위한 예시로요. 더 많은 유연성과 다양성을 원한다면, 구체적인 예시보다 설명적인 예시를 사용할 거예요.

- 한 가지만 덧붙이자면, Claude.ai에서 사용할 프롬프트를 쓸 때는 한 번 맞을 때까지 반복하는 거예요. 그러면 끝이에요. 반면에 대부분의 엔터프라이즈 프롬프트는 이걸 백만 번이나 천만 번, 아니면 억 번 사용할 거잖아요. 그래서 들이는 정성과 생각은 사용될 수 있는 모든 범위의 방식과 입력 데이터의 범위에 대해 테스트하는 거예요.

## 핵심 요약

- **연구용**: 다양성 추구, 예시 적게 사용, 설명적 예시 선호
- **엔터프라이즈**: 신뢰성과 일관성 중시, 예시 많이 사용
- **일반 채팅**: 한 번 성공하면 끝
- 엔터프라이즈는 백만 번 사용되므로 전체 입력 범위 테스트 필요
