# 코딩 에이전트 101: 실제로 일을 완수하는 기술

[원본 링크](https://devin.ai/agents101)

"코딩 에이전트는 마법이 아니지만, 우리가 가진 것 중 가장 마법에 가깝습니다." 2025년 6월 Cognition 팀이 작성한 이 가이드는 자율 에이전트를 도입하려는 엔지니어를 위한 제품 중립적이고 실용적인 기술 조언을 담은 15분 분량의 글입니다.

개발자 도구는 자동완성(10년 전)에서 코파일럿(4년 전), 생성형 챗봇(2년 전)을 거쳐 오늘날의 자율 에이전트로 진화했습니다. 자율 에이전트는 "초기 설명부터 최종 풀 리퀘스트까지 인간 개입을 최소화하며" 작업을 처리합니다.

## 프롬프팅 기본

### 무엇을 원하는지뿐 아니라 *어떻게* 하길 원하는지 말하세요

에이전트를 의사결정이 불안정한 주니어 코딩 파트너처럼 대하세요. 복잡한 작업에서는 선호하는 접근 방식을 미리 설명하세요. 성공률이 높아지고, 의도한 방법을 이미 이해하므로 리뷰 시간도 줄어듭니다.

**제시된 예시:** "유닛 테스트 추가해줘"라고 요청하는 대신, 어떤 기능을 테스트할지 명시하고, 중요한 엣지 케이스를 식별하고, 모킹 요구사항을 명확히 하세요.

### 에이전트에게 시작 지점을 알려주세요

본인이라면 어디서 시작할지 생각해보세요. 구체적인 파일명이 없더라도 리포지토리, 관련 문서, 주요 컴포넌트를 언급하면 낭비되는 노력을 줄일 수 있습니다.

**예시:** "우리 코드에 Google 모델 지원을 추가해주세요. [여기]에서 최신 문서를 확인하고 model groups 디렉토리에 새 구현 파일을 만들어주세요."

### 방어적 프롬프팅을 연습하세요

새로운 인턴에게 설명한다고 생각하세요. 혼란스러울 수 있는 부분을 예상하고, 모호함을 없애기 위해 미리 명확히 하세요.

**예시:** "새 유닛 테스트를 통과하도록 검색 모듈의 C++ 바인딩을 수정해주세요. 주의하세요, 테스트 전에 코드를 변경할 때마다 바인딩을 다시 컴파일해야 합니다."

### CI, 테스트, 타입, 린터 접근 권한을 주세요

강력한 피드백 루프가 있으면 에이전트 성능이 극적으로 개선됩니다. 타입 체커, 린터, 유닛 테스트를 활용하세요. 타입 없는 Python보다 타입 있는 Python을, JavaScript보다 TypeScript를 선호하세요. 에이전트에게 일반적인 검사 실행 방법을 가르치고 필요한 접근 권한을 제공하세요.

### 당신의 전문성을 활용하세요

인간의 감독은 여전히 필수입니다. 코드 정확성의 책임은 당신에게 있습니다. "도구가 아무리 정교해지더라도 소유권과 검증은 인간 엔지니어의 핵심 책임으로 남을 것입니다."

## 워크플로우에서 에이전트 활용하기

### 새로운 작업을 즉시 처리하세요

요청이 집중을 방해하도록 두지 마세요. 자율 에이전트에게 조사나 변경 구현을 위임하면 본 업무에 집중할 수 있습니다.

**예시:** 팀들이 버그 수정이나 사소한 업데이트를 논의할 때 Slack에서 @Devin을 태그합니다.

### 이동 중에도 코딩하세요

Slack 앱이나 전용 애플리케이션을 통해 모바일로 접근하면, 출퇴근 중이나 여행 중에도 긴급한 문제를 처리할 수 있습니다.

### 잡무를 넘기세요

커밋 이분법 찾기나 문서 업데이트 같은 반복 작업을 위임하면 창의적인 작업에 시간을 쓸 수 있습니다.

**예시:** 엔지니어가 변경 사항을 배포하면 에이전트가 관련 문서와 사용자 대면 문구를 업데이트합니다.

### 분석 마비를 건너뛰세요

아키텍처 접근 방식을 결정할 때 에이전트가 여러 옵션을 구현하도록 하세요. 구체적인 예시가 있으면 거절된 솔루션에 감정 상처 없이 결정할 수 있습니다.

**예시:** 텍스트 에디터 라이브러리를 선택할 때 에이전트가 두 옵션을 모두 구현하여 결과를 비교했습니다.

### 미리보기 배포를 설정하세요

각 PR마다 자동으로 미리보기 배포가 생성되도록 CI/CD 파이프라인을 구성하면, 특히 프론트엔드 작업에서 즉시 라이브 URL을 확인할 수 있습니다.

## 대규모 티켓 위임하기

이 섹션은 중규모에서 대규모 작업(보통 1-6시간 분량)을 다룹니다. 자율 에이전트는 분 단위가 아닌 시간 단위를 되찾아주어 가장 높은 ROI를 제공합니다.

### 첫 번째 초안을 자동화하세요

상당한 규모의 작업에서는 에이전트로 초기 PR 초안을 만드세요. 성공의 열쇠는 원하는 접근 방식을 미리 명확하게 전달하는 것입니다. 주니어 개발자를 안내하는 아키텍트처럼 기능합니다.

여러 분야에 걸친 비유:
- **저널리즘:** 기자가 정보를 수집하고 초안을 작성합니다. 편집자가 검토, 팩트체크, 마무리합니다.
- **레스토랑:** 라인 쿡이 재료를 준비하고 예비 요리를 만듭니다. 수석 셰프가 간을 보고 조정합니다.
- **코딩:** 자율 에이전트가 초기 계획 기반으로 작업을 시작하고 초안 솔루션을 만듭니다. 인간 개발자가 검토하고, 피드백을 주고, 개선합니다.

현실적 기대치는 완전한 자동화가 아닌 약 80% 시간 절약입니다. 검증과 최종 품질 보증에는 인간 전문성이 여전히 필수입니다.

### PRD를 공동 개발하세요

복잡하거나 모호하게 정의된 작업은 에이전트와 협력해 상세한 계획을 세우세요. "인증 시스템이 어떻게 작동하나요?" 또는 "어떤 서비스가 영향을 받을 수 있나요?" 같은 탐색 질문으로 시작하세요.

일부 에이전트는 기존 코드를 읽고 탐색하는 데 집중하는 전용 계획 모드를 제공합니다. 특화된 코드베이스 검색 도구로 빠르게 인사이트를 얻을 수 있습니다.

### 체크포인트를 설정하세요

여러 코드베이스에 걸친 다중 파트 작업에서는 명확한 체크포인트를 설정하세요:
계획 → 청크 구현 → 테스트 → 수정 → 체크포인트 검토 → 다음 청크

특히 여러 레이어(데이터베이스, 백엔드, 프론트엔드)에 걸친 복잡한 기능은 중요한 단계 후에 명시적으로 일시 정지를 요청하세요.

**예시:** "이 기능을 데이터베이스, 백엔드, 여러 프론트엔드 인터페이스에 걸쳐 구현해주세요. 먼저 필요한 데이터베이스 스키마 변경을 계획하고, 완료되면 알려주세요. 마이그레이션을 적용하겠습니다..." 그 후 백엔드 변경과 프론트엔드 구현을 순차적으로 요청합니다.

### 스스로 작업을 검증하도록 가르치세요

피드백을 줄 때 테스팅 프로세스를 설명하면 에이전트가 향후 작업을 독립적으로 검증할 수 있습니다. 자주 반복되는 테스팅 패턴은 에이전트의 영구 지식 베이스에 추가하세요.

### AI가 자주 수정하는 영역의 테스트 커버리지를 높이세요

에이전트는 모든 시나리오를 철저하게 대화형으로 테스트할 수 없으므로, 자주 수정되는 영역의 테스트 커버리지를 강화해 신뢰도를 높이세요.

**예시:** 한 팀이 AI가 구현을 Python에서 C로 번역하기 전에 핵심 섹션의 유닛 테스트를 강화했습니다.

## 워크플로우 자동화

### 가장 반복적인 작업의 단축키를 만드세요

엔지니어링 팀은 자동화에 적합한 일상적인 작업을 자주 접합니다:
- 기능 플래그 제거
- 의존성 업그레이드
- 새 기능 PR에 테스트 수정 및 추가

경험 많은 엔지니어가 이런 시나리오에 반복 사용할 수 있는 견고한 프롬프트 템플릿을 만듭니다.

**예시:** 한 고객은 새 기능을 개발할 때마다 유닛 테스트 작성 전용 에이전트 세 개를 자동으로 트리거합니다.

### 지능형 코드 리뷰 및 적용

자율 에이전트는 리포지토리 기능을 인덱싱하면 특화된 도구보다 더 정확한 인사이트를 제공합니다.

**예시:** Cognition 팀은 엔지니어들이 자주 범하는 실수 목록을 코드베이스에 유지하고, 모든 PR에서 이 문제들을 확인하는 에이전트를 실행합니다.

### 인시던트와 알림에 연결하세요

API나 CLI 명령을 통해 특정 이벤트에 대응하여 에이전트가 자동 트리거되도록 설정하세요. 서드파티 오류 로그를 수집하는 MCP와 함께 사용하면 특히 효과적입니다.

**중요 참고:** 프로덕션 서비스의 AI 디버깅 기술은 아직 제한적입니다. AI에게 버그를 처음부터 끝까지 수정하도록 요청하기보다 가장 의심스러운 오류와 변경 사항을 표시하도록 요청하세요.

## 커스터마이징 및 성능 개선

### 환경 설정

언어 버전, 패키지 의존성, 자동화된 검사를 포함해 에이전트 설정을 팀과 정확히 동일하게 맞추세요. Pre-commit을 설치하고, 환경 구성(시크릿, 언어 버전, 가상 환경, 브라우저 로그인)을 자동으로 불러오게 하세요.

**예시:** 테스트 중 수동 인증의 번거로움을 없애려면 브라우저 로그인을 미리 설정하세요.

### 커스텀 CLI 도구와 MCP 구축하기

MCP는 에이전트를 외부 도구에 연결하기 위해 빠르게 설정하고 실험할 수 있습니다. 하지만 단순한 CLI 스크립트도 간과하지 마세요. Linear 티켓 정보를 가져오거나 개발 환경을 재시작하는 등 신뢰할 수 있는 워크플로우 스크립트를 고려하세요.

**예시:** 한 고객이 첫 번째 실패 테스트만 표시하는 CLI 도구를 만들어 에이전트가 상세한 오류 정보에 집중하게 했고, 더 높은 성공률과 빠른 완료 시간을 달성했습니다.

### 에이전트의 지식 베이스에 추가하세요

에이전트가 흔한 실수를 하면 피드백을 지식 베이스에 문서화하세요. 프레임워크 가이드라인만 주지 말고, 전체 프로젝트 아키텍처, 작업 유형별 테스팅 접근법, 중요한 명령어, 도구 권장 사항을 포함하세요.

**예시:** 프론트엔드와 백엔드의 모든 보일러플레이트 위치를 포함한 새 서비스 라우트 추가 절차를 제공하면 쉽게 위임할 수 있습니다.

## 실용적 고려사항

### 자율 에이전트의 한계

**제한된 디버깅 기술:** 많은 버그는 현재 AI 에이전트 능력을 넘어서는 데이터베이스 및 로그 접근과 디버깅 전문성을 요구합니다. 완전한 디버깅 대신 가능한 근본 원인 목록을 요청하세요. 원인을 파악하면 에이전트가 효과적으로 수정합니다.

**부족한 세밀한 시각적 추론:** 현재 모델은 디자인 스크린샷이나 피그마 목업을 정밀하게 매칭하는 시각적 추론 능력이 부족합니다. 코드 수준의 비주얼에서 가장 신뢰성 있게 작동합니다. 시각적 일관성을 위해 재사용 가능한 컴포넌트가 있는 좋은 디자인 시스템을 사용하세요.

**지식 컷오프:** 새 라이브러리 작업 시 에이전트에게 최신 문서를 명시적으로 안내하세요. 대부분의 에이전트는 사전 훈련된 모델의 지식 컷오프로 인해 오래된 패턴을 가정합니다.

### 시간 관리 및 손실 최소화

2025년 에이전트 결과는 변동성이 있습니다. 성공적인 결과를 최대화하고 낭비되는 시간과 토큰을 최소화하는 것이 중요합니다.

**더 빨리 손절하세요:** 새로운 사용자들은 에이전트가 방향을 잃어도 상호작용을 성공시키려고 집착하는 경향이 있습니다. 에이전트가 지시를 무시하거나 제자리를 맴돌면 중단하거나 직접 인수하세요. 추가 메시지가 필요하다면 보통 단순한 실수가 아니라 작업 복잡도가 에이전트 능력을 초과한 것입니다.

**실험을 다양화하세요:** 다양한 프롬프트와 접근 방식을 시도하세요. 에이전트가 자연스럽게 잘하는 작업에 집중하고, 성공하지 못하는 유형은 손절하세요.

**진전이 없으면 새로 시작하세요:** 모든 지시를 포함한 새 에이전트로 시작하는 것이 망가진 환경을 수정하는 것보다 빠른 경우가 많습니다. 에이전트의 문제 수정 능력은 처음부터 새 코드를 작성하는 능력보다 떨어집니다.

### 보안 및 권한

**에이전트용 계정을 만드세요:** 안전한 사이트 테스트를 위해 일회용 이메일을 사용하세요. 클라우드 리소스에 접근한다면 커스텀 IAM 역할을 만드세요.

**개발/스테이징 환경을 제공하세요:** 에이전트는 팀 엔지니어와 동일한 테스트 설정을 사용해야 하며, 프로덕션 접근은 완전히 피해야 합니다. 원격 에이전트는 완전히 격리된 테스트 환경에서 실행할 수 있습니다.

**읽기 전용 API 키:** 가능하면 읽기 전용 접근을 제공하세요. 외부 서비스와 상호작용하는 스크립트는 사람이 직접 실행하는 것이 좋습니다.

## 결론: 큰 변화가 다가온다

소프트웨어 엔지니어는 여전히 필수입니다. "코딩 에이전트가 점점 스마트해지고 능력이 커져도 깊은 기술 전문성과 코드베이스에 대한 깊은 이해는 여전히 귀중합니다." 자동화가 영향력을 증폭시킬수록 프로젝트와 시스템의 진정한 소유권이 더욱 중요해집니다.

엔지니어들은 깊은 이해와 신중한 판단을 유지하면서 여러 시스템을 감독하도록 점점 더 기대됩니다. 자동화가 병렬 작업 처리를 가능하게 하면서 이 능력은 선택이 아닌 필수가 됩니다. Cognition 팀은 팀들이 진화하는 소프트웨어 개발 환경에서 번창할 수 있도록 이런 인사이트를 공유합니다.

## 핵심 요약

- 코딩 에이전트는 자동완성에서 자율 에이전트로 진화한 개발자 도구의 최신 형태
- 에이전트에게 무엇을 원하는지뿐 아니라 어떻게 하길 원하는지 설명하면 성공률 향상
- 방어적 프롬프팅으로 혼란 가능성을 미리 차단하고, CI/테스트/린터 접근 권한 제공이 중요
- 대규모 작업에서 80% 시간 절약 가능하지만, 검증과 소유권은 여전히 인간의 몫
- 반복 작업을 자동화하고, 에이전트의 지식 베이스에 프로젝트 컨텍스트 추가 권장
- 에이전트의 한계(디버깅, 시각적 추론, 지식 컷오프) 인지하고 적절히 활용
- 진전 없으면 빨리 손절하고 새로 시작하는 것이 효과적
