# MCP 레지스트리 소개

MCP 레지스트리가 프리뷰로 공개됐습니다.

https://registry.modelcontextprotocol.io에서 MCP 레지스트리를 출시하게 되어 기쁩니다. MCP 레지스트리는 공개 MCP 서버를 쉽게 찾고 구현할 수 있도록 돕는 오픈 카탈로그이자 API입니다. 서버 배포와 검색 방식을 표준화해, 서버가 더 널리 사용되게 하면서 클라이언트가 쉽게 연결할 수 있게 합니다.

MCP 프로젝트의 일환으로, MCP 레지스트리와 상위 OpenAPI 명세는 오픈소스로 제공되어 누구나 호환 가능한 서브레지스트리를 구축할 수 있습니다.

## 배경

2025년 2월, MCP 창시자인 David Soria Parra와 Justin Spahr-Summers가 PulseMCP와 Goose 팀에게 중앙화된 커뮤니티 레지스트리 구축을 요청하면서 이 프로젝트는 커뮤니티 주도의 협업으로 시작됐습니다. PulseMCP의 레지스트리 관리자 Tadas Antanavicius가 Block의 Alex Hancock과 협력하여 초기 작업을 주도했습니다. 이후 GitHub MCP 책임자인 레지스트리 관리자 Toby Padilla가 합류했고, 최근에는 Anthropic의 Adam Jones가 레지스트리 관리자로 합류했습니다.

## 아키텍처

레지스트리의 핵심 아키텍처 결정 중 하나는 연합(federation)을 채택한 것입니다. 업스트림 MCP 레지스트리가 유일한 레지스트리는 아닙니다. 대신, 서브레지스트리(공개 또는 비공개)가 수집, 보강, 미러링할 수 있는 공개 MCP 서버 메타데이터의 공식 소스 역할을 합니다.

### 설계 원칙

MCP 레지스트리는 다음과 같은 핵심 설계 원칙을 따릅니다:

- **단일 진실 공급원(Single Source of Truth)**: 공개 MCP 서버의 권위 있는 메타데이터 저장소
- **벤더 중립성**: 특정 서버나 조직을 우대하지 않음
- **업계 보안 표준**: 기존 패키지 레지스트리의 보안 방식 활용
- **재사용성**: 비공개/내부 레지스트리를 지원하는 재사용 가능한 API 설계
- **점진적 개선**: MVP로 시작하여 향후 기능을 위한 기반 구축

### 작동 방식

MCP 레지스트리는 메타레지스트리입니다. 패키지에 대한 메타데이터를 호스팅하지만, 패키지 코드나 바이너리는 호스팅하지 않습니다. 구체적으로, MCP 서버에 대한 메타데이터와 해당 서버가 호스팅되는 위치(npm, PyPI, NuGet, Docker Hub 등)에 대한 참조를 저장하지만, 실제 소스 코드나 패키지는 호스팅하지 않습니다.

서버 관리자는 각 MCP 서버를 설명하는 메타데이터(엔드포인트, 기능, 버전 관리 및 관련 필드)를 게시합니다. 레지스트리는 신뢰할 수 있는 카탈로그를 유지하기 위해 최소한의 검증(예: 네임스페이스 고유성, 스키마 적합성)을 수행합니다.

registry.modelcontextprotocol.io의 읽기 전용 API는 서버 목록(예: GET /v0/servers)을 노출하여, 클라이언트와 수집기가 카탈로그를 최신 상태로 쉽게 유지할 수 있습니다.

## 서브레지스트리

각 MCP 클라이언트와 연결된 "MCP 마켓플레이스"와 같은 공개 서브레지스트리는 업스트림 MCP 레지스트리에서 수집한 데이터를 자유롭게 보강하고 확장할 수 있습니다. MCP 사용자마다 요구사항이 다르므로, 각 마켓플레이스가 자신만의 방식으로 사용자를 지원합니다.

비공개 서브레지스트리는 엄격한 프라이버시와 보안 요구사항을 가진 기업 내에 존재하며, MCP 레지스트리는 이러한 기업들이 기반으로 삼을 수 있는 단일 업스트림 데이터 소스를 제공합니다. 연관된 SDK와 도구를 생태계 전반에서 공유할 수 있도록 이러한 비공개 구현과 API 스키마를 공유하는 것을 목표로 합니다.

### 서브레지스트리 기능

서브레지스트리는 다음 기능을 제공하여 레지스트리 생태계에 가치를 더합니다:

- **큐레이션**: 특정 커뮤니티나 사용 사례에 맞는 서버 필터링
- **평점**: 사용자 평점 및 다운로드 통계 추가
- **보안**: 보안 스캐닝 및 취약점 검사 구현
- **엔터프라이즈**: 기업 사용자를 위한 내부 서버 레지스트리 제공

### 서브레지스트리 작동 방식

서브레지스트리(GitHub나 PulseMCP 같은 플랫폼이 운영)는 업스트림 레지스트리에서 전체 데이터셋을 주기적으로(예: 하루에 한 번) 가져옵니다. 각 서브레지스트리는 자체 로직을 적용합니다:

- **심사/필터링**: 플랫폼의 보안 또는 성능 기준을 충족하지 않는 서버를 제외할 수 있음
- **보강**: 저장소 스타 수와 같은 플랫폼별 메타데이터 추가
- **캐싱**: 클라이언트를 위한 높은 가용성과 낮은 지연시간을 보장하기 위해 데이터 캐싱

레지스트리는 주로 서브레지스트리(Smithery, PulseMCP, Docker Hub, Anthropic, GitHub 등)가 프로그래밍 방식으로 사용하도록 설계됐습니다. 현재 개별 클라이언트나 최종 사용자용이 아니며, 사용자는 서브레지스트리를 이용해야 합니다.

## API 접근

MCP 레지스트리 API는 서버 메타데이터에 프로그래밍 방식으로 접근할 수 있게 합니다:

- **프로덕션**: https://registry.modelcontextprotocol.io
- **스테이징**: https://staging.registry.modelcontextprotocol.io

### API 엔드포인트

curl 명령어로 레지스트리 API와 상호작용할 수 있습니다:

```bash
# 서버 목록 조회
curl "https://registry.modelcontextprotocol.io/v0/servers?limit=10"

# 특정 서버 검색
curl "https://registry.modelcontextprotocol.io/v0/servers?search=filesystem"

# 특정 서버 상세 정보 조회
curl "https://registry.modelcontextprotocol.io/v0/servers/{server-id}"
```

/v0/servers 엔드포인트는 GET 메서드로 사용 가능한 모든 서버 목록을 조회합니다. 이 엔드포인트는 제한이 있으며 커서 기반 페이지네이션을 사용합니다.

## 시작하기

### 서버 관리자용

서버 관리자는 MCP 레지스트리에 서버 추가하기 가이드를 따라 서버를 추가할 수 있습니다. 네임스페이스 게시를 위해서는 소유권을 증명해야 합니다:

- GitHub 기반 네임스페이스(예: `io.github.domdomegg/my-cool-mcp`)의 경우, 해당 사용자로 GitHub에 로그인하거나 해당 사용자의 저장소에서 GitHub Action을 실행해야 합니다
- 도메인 기반 네임스페이스(예: `me.adamjones/my-cool-mcp`)의 경우, DNS 또는 HTTP 챌린지를 통해 소유권을 증명해야 합니다

MCP 레지스트리 퍼블리셔 도구는 다음 명령어를 제공합니다:

- `init`: server.json 파일 템플릿 생성
- `login`: 레지스트리 인증
- `logout`: 저장된 인증 정보 삭제
- `publish`: server.json을 레지스트리에 게시

### 클라이언트 관리자용

클라이언트 관리자는 MCP 레지스트리 데이터 접근하기 가이드를 따라 서버 데이터에 접근할 수 있습니다.

## 커뮤니티 모더레이션

시스템은 모더레이션 가이드라인을 위반하는 서버 신고를 지원하며, 관리자는 문제가 있는 항목을 차단 목록에 추가할 수 있습니다.

## 프리뷰 상태

이 프리뷰를 통해 정식 출시 전에 사용자 경험을 개선하려 합니다. 최종 릴리스 전에 호환성을 깨는 변경이 있을 수 있습니다.

2025년 10월 기준으로, 레지스트리 API는 API 프리즈(v0.1)에 들어갔습니다. 앞으로 한 달 이상 API는 호환성을 깨는 변경 없이 안정적으로 유지되어, 통합 개발자들이 안심하고 지원을 구현할 수 있습니다.

## 감사의 말

MCP 레지스트리 교차 기업 워킹 그룹에는 수석 관리자 David Soria Parra와 Anthropic, GitHub, PulseMCP의 레지스트리 관리자들, 그리고 Block과 Microsoft의 기여자들이 포함됩니다.

이 프로젝트는 PulseMCP, GitHub, Block, Anthropic 및 AI 개발 생태계 전반의 여러 조직이 참여한 2025년 2월의 커뮤니티 협업에서 시작됐습니다.

## 핵심 요약

- MCP 레지스트리는 공개 MCP 서버를 쉽게 찾고 구현할 수 있도록 돕는 오픈 카탈로그 및 API
- 연합(federation) 아키텍처를 채택하여 공개 및 비공개 서브레지스트리가 데이터를 수집, 보강, 미러링 가능
- 메타레지스트리로서 메타데이터만 저장하며, 실제 코드나 패키지는 호스팅하지 않음
- 서브레지스트리는 큐레이션, 평점, 보안, 엔터프라이즈 기능을 제공하여 생태계에 가치 추가
- 현재 프리뷰 상태이며 2025년 10월 기준 API 프리즈(v0.1)에 진입
