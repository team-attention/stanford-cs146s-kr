# Anthropic 팀들이 Claude Code를 활용하는 방법

[원본 링크](https://www-cdn.anthropic.com/58284b19e702b49db9302d5b6f135ad8871e7658.pdf)

Anthropic 내부 팀들이 Claude Code로 업무 방식을 혁신하고 있습니다. 개발자와 비기술 직원 모두 복잡한 프로젝트를 수행하고, 작업을 자동화하며, 그동안 생산성을 제한했던 기술 격차를 해소할 수 있게 되었습니다.

Claude Code 파워 유저들과 인터뷰를 통해 각 부서의 Claude Code 활용법, 업무에 미친 영향, 도입을 고려하는 다른 조직을 위한 팁을 수집했습니다.

## 데이터 인프라 팀의 Claude Code 활용

데이터 인프라 팀은 회사 전체 팀을 위한 모든 비즈니스 데이터를 조직합니다. 이 팀은 Claude Code로 일상적인 데이터 엔지니어링 작업을 자동화하고, 복잡한 인프라 문제를 해결하며, 기술/비기술 팀원 모두가 독립적으로 데이터에 접근하고 조작할 수 있는 문서화된 워크플로우를 만듭니다.

### 주요 Claude Code 활용 사례

**스크린샷을 활용한 쿠버네티스 디버깅**

쿠버네티스 클러스터가 다운되어 새 파드를 스케줄링하지 못했을 때, 팀은 Claude Code로 문제를 진단했습니다. 대시보드 스크린샷을 Claude Code에 입력하자 Google Cloud UI를 메뉴별로 안내하며 파드 IP 주소 고갈 경고를 찾아냈습니다. Claude Code가 새 IP 풀을 생성하고 클러스터에 추가하는 정확한 명령어를 제공해 네트워킹 전문가 없이 문제를 해결했습니다.

**재무팀을 위한 일반 텍스트 워크플로우**

팀은 재무팀 직원들에게 데이터 워크플로우를 일반 텍스트 파일로 작성한 후 Claude Code에 로드하여 완전히 자동화된 실행을 받는 방법을 알려주었습니다. 코딩 경험이 전혀 없는 직원도 "이 대시보드를 쿼리하고, 정보를 가져오고, 이 쿼리들을 실행하고, Excel 결과물을 생성해줘" 같은 단계를 설명하면 Claude Code가 날짜 같은 필수 입력값을 요청하며 전체 워크플로우를 실행합니다.

**신입 사원을 위한 코드베이스 탐색**

새 데이터 사이언티스트가 팀에 합류하면 거대한 코드베이스 탐색에 Claude Code를 사용하도록 안내받습니다. Claude Code가 Claude.md 파일(문서)을 읽고, 특정 작업 관련 파일을 찾고, 데이터 파이프라인 의존성을 설명하고, 어떤 업스트림 소스가 대시보드에 데이터를 공급하는지 신입 사원이 이해하도록 돕습니다. 기존의 데이터 카탈로그와 검색 도구를 대체합니다.

**세션 종료 시 문서 업데이트**

팀은 각 작업이 끝날 때마다 Claude Code에게 완료된 작업 세션을 요약하고 개선점을 제안하도록 요청합니다. Claude Code가 실제 사용 경험을 바탕으로 Claude.md 문서와 워크플로우 지침을 개선하는 지속적인 개선 루프가 만들어져 후속 작업이 더 효과적으로 진행됩니다.

**여러 인스턴스에서 병렬 작업 관리**

오래 걸리는 데이터 작업 시 팀은 다른 프로젝트의 다른 저장소에서 여러 Claude Code 인스턴스를 엽니다. 각 인스턴스가 전체 컨텍스트를 유지하므로 몇 시간 또는 며칠 후에 돌아와도 Claude Code가 무엇을 하고 있었고 어디까지 진행했는지 정확히 기억해 컨텍스트 손실 없이 진정한 병렬 워크플로우 관리가 가능합니다.

### 팀에 미친 영향

- **전문 지식 없이 인프라 문제 해결**: 일반적으로 시스템이나 네트워킹 팀원을 불러야 하는 쿠버네티스 클러스터 문제를 Claude Code로 진단하고 정확한 수정 방법을 제공받아 해결했습니다.
- **온보딩 가속화**: 새 데이터 분석가와 팀원이 광범위한 안내 없이도 복잡한 시스템을 빠르게 이해하고 의미 있게 기여할 수 있습니다.
- **지원 워크플로우 강화**: 훨씬 더 많은 데이터 볼륨을 처리하고 사람이 수동으로 검토하기 불가능한 이상 징후(예: 200개 대시보드 모니터링)를 식별할 수 있습니다.
- **팀 간 셀프 서비스 실현**: 코딩 경험이 없는 재무팀도 복잡한 데이터 워크플로우를 독립적으로 실행할 수 있습니다.

### 데이터 인프라 팀의 핵심 팁

1. **상세한 Claude.md 파일 작성**: Claude.md 파일에 워크플로우, 도구, 기대사항을 더 잘 문서화할수록 Claude Code의 성능이 향상됩니다. 기존 패턴이 있을 때 새 데이터 파이프라인 설정 같은 일상 작업에서 Claude Code가 탁월한 성과를 냅니다.

2. **민감한 데이터에는 CLI 대신 MCP 서버 사용**: BigQuery CLI 대신 MCP 서버를 사용해 Claude Code의 접근 범위를 더 잘 보안 통제하길 권장합니다. 특히 로깅이 필요하거나 프라이버시 우려가 있는 민감한 데이터를 처리할 때 유용합니다.

3. **팀 사용 세션 공유**: 팀원들이 서로의 Claude Code 워크플로우를 시연하는 세션을 개최했습니다. 모범 사례가 퍼지고 혼자서는 발견하지 못했을 다양한 사용 방법을 알게 되었습니다.

## 제품 개발 팀의 Claude Code 활용

Claude Code 팀은 자체 제품을 사용해 Claude Code 업데이트를 구축하고, 제품의 엔터프라이즈 기능과 에이전틱 루프 기능을 확장합니다.

### 주요 Claude Code 활용 사례

**자동 승인 모드를 활용한 빠른 프로토타이핑**

엔지니어들은 "자동 승인 모드"(shift+tab)를 활성화하고 Claude가 코드를 작성하고, 테스트를 실행하고, 지속적으로 반복하는 자율 루프를 설정해 빠른 프로토타이핑에 Claude Code를 사용합니다. 익숙하지 않은 추상적인 문제를 Claude에게 주고 자율적으로 작업하게 한 다음, 80% 완성된 솔루션을 검토한 후 최종 다듬기를 직접 수행합니다. 팀은 깨끗한 git 상태에서 시작하고 정기적으로 체크포인트를 커밋해 Claude가 잘못된 방향으로 갈 경우 쉽게 되돌릴 수 있도록 강조합니다.

**핵심 기능을 위한 동기식 코딩**

애플리케이션의 비즈니스 로직을 건드리는 더 중요한 기능의 경우, 팀은 Claude Code와 동기적으로 작업하며 구체적인 구현 지침이 포함된 상세한 프롬프트를 제공합니다. Claude가 반복적인 코딩 작업을 처리하는 동안 코드 품질, 스타일 가이드 준수, 적절한 아키텍처를 보장하기 위해 실시간으로 프로세스를 모니터링합니다.

**Vim 모드 구축**

가장 성공적인 비동기 프로젝트 중 하나는 Claude Code용 Vim 키 바인딩 구현이었습니다. 우선순위가 아니었음에도 Claude에게 전체 기능을 만들어달라고 요청했고, 최종 구현의 약 70%가 Claude의 자율 작업에서 나왔으며 완료까지 몇 번의 반복만 필요했습니다.

**테스트 생성 및 버그 수정**

기능 구현 후 포괄적인 테스트를 작성하고 PR 리뷰에서 발견된 간단한 버그 수정을 처리하는 데 Claude Code를 사용합니다. GitHub Actions 통합을 활용해 Claude가 포맷팅 문제나 함수 이름 변경 같은 PR 코멘트를 자동으로 처리하도록 합니다.

**코드베이스 탐색**

익숙하지 않은 코드베이스(예: 모노레포 또는 API 측)에서 작업할 때 팀은 Claude Code로 시스템 작동 방식을 빠르게 이해합니다. Slack 응답을 기다리는 대신 Claude에게 직접 설명과 코드 참조를 요청해 컨텍스트 전환에 드는 상당한 시간을 절약합니다.

### 팀에 미친 영향

- **더 빠른 기능 구현**: Claude가 자율적으로 70%의 코드를 작성해 Vim 모드 같은 복잡한 기능을 성공적으로 구현했습니다.
- **개발 속도 향상**: 구현 세부 사항에 막히지 않고 기능을 빠르게 프로토타이핑하고 아이디어를 반복할 수 있습니다.
- **자동화된 테스트를 통한 코드 품질 향상**: Claude가 포괄적인 테스트를 생성하고 일상적인 버그 수정을 처리해 수작업 노력을 줄이면서 높은 표준을 유지합니다.
- **더 나은 코드베이스 탐색**: 팀원들이 동료의 응답을 기다리지 않고 모노레포의 익숙하지 않은 부분을 빠르게 이해할 수 있습니다.

### Claude Code 팀의 핵심 팁

1. **자기 충족적 루프 생성**: Claude가 빌드, 테스트, 린트를 자동으로 실행해 자체 작업을 검증하도록 설정하세요. Claude가 더 오래 자율적으로 작업하고 자신의 실수를 잡을 수 있습니다. 특히 코드를 작성하기 전에 테스트를 먼저 생성하도록 요청할 때 효과적입니다.

2. **작업 분류 직관 개발**: 비동기적으로 잘 작동하는 작업(주변부 기능, 프로토타이핑)과 동기적 감독이 필요한 작업(핵심 비즈니스 로직, 중요한 수정)을 구분하는 법을 배우세요. 제품 주변부의 추상적인 작업은 "자동 승인 모드"로 처리할 수 있지만, 핵심 기능은 더 면밀한 감독이 필요합니다.

3. **명확하고 상세한 프롬프트 작성**: 컴포넌트 이름이나 기능이 비슷할 때 요청을 매우 구체적으로 하세요. 프롬프트가 더 좋고 상세할수록 코드베이스의 잘못된 부분에 예상치 못한 변경 없이 Claude가 독립적으로 작업하도록 신뢰할 수 있습니다.

## 보안 엔지니어링 팀의 Claude Code 활용

보안 엔지니어링 팀은 소프트웨어 개발 라이프사이클 보안, 공급망 보안, 개발 환경 보안에 집중합니다. 코드 작성과 디버깅에 Claude Code를 광범위하게 사용합니다.

### 주요 Claude Code 활용 사례

**복잡한 인프라 디버깅**

인시던트 작업 시 Claude Code에 스택 트레이스와 문서를 입력하고 코드베이스 전체의 제어 흐름을 추적하도록 요청합니다. 프로덕션 문제 해결 시간이 크게 단축되어 일반적으로 수동 코드 스캔에 10-15분 걸리던 문제를 약 5분 만에 이해할 수 있습니다.

**Terraform 코드 리뷰 및 분석**

보안 승인이 필요한 인프라 변경 시 Terraform 플랜을 Claude Code에 복사해 "이게 뭘 하는 거야? 후회할 일이 생길까?"라고 질문합니다. 더 촘촘한 피드백 루프가 만들어지고 보안 팀이 인프라 변경을 빠르게 검토하고 승인할 수 있어 개발 프로세스의 병목이 줄어듭니다.

**문서 종합 및 런북**

Claude Code에 여러 문서 소스를 입력해 마크다운 런북, 문제 해결 가이드, 개요를 만들도록 합니다. 이렇게 압축된 문서를 실제 문제 디버깅의 컨텍스트로 사용해 전체 지식 베이스를 검색하는 것보다 더 효율적인 워크플로우를 만듭니다.

**테스트 주도 개발 워크플로우**

이전의 "설계 문서 → 대충 짠 코드 → 리팩토링 → 테스트 포기" 패턴 대신, Claude Code에 의사 코드를 요청하고 테스트 주도 개발을 안내하며 막힐 때 주기적으로 확인해 방향을 잡아줍니다. 그 결과 더 안정적이고 테스트 가능한 코드가 나옵니다.

**컨텍스트 전환 및 프로젝트 온보딩**

"dependant"(보안 승인 워크플로우용 웹 애플리케이션) 같은 기존 프로젝트에 기여할 때 Claude Code로 코드베이스에 저장된 마크다운 형식의 명세를 작성, 검토, 실행합니다. 몇 주가 아닌 며칠 만에 의미 있는 기여가 가능합니다.

### 팀에 미친 영향

- **인시던트 해결 시간 단축**: 일반적으로 수동 코드 스캔에 10-15분 걸리던 인프라 디버깅이 약 5분으로 줄었습니다.
- **보안 검토 주기 개선**: 보안 승인을 위한 Terraform 코드 검토가 훨씬 빨라져 보안 팀 승인을 기다리는 개발자 블로킹이 해소되었습니다.

## 추론(Inference) 팀의 Claude Code 활용

추론 팀은 Claude가 프롬프트를 읽고 응답을 생성하는 동안 정보를 저장하는 메모리 시스템을 관리합니다. 특히 머신러닝이 처음인 팀원들이 Claude Code를 광범위하게 사용해 지식 격차를 해소하고 업무를 가속화합니다.

### 주요 Claude Code 활용 사례

**코드베이스 이해 및 온보딩**

팀은 복잡한 코드베이스에 합류할 때 아키텍처를 빠르게 이해하기 위해 Claude Code에 크게 의존합니다. GitHub 저장소를 수동으로 검색하는 대신 Claude에게 어떤 파일이 특정 기능을 호출하는지 질문하면 동료에게 묻거나 수동 검색하는 것보다 몇 초 만에 결과를 얻습니다.

**엣지 케이스를 포함한 유닛 테스트 생성**

핵심 기능을 작성한 후 Claude에게 포괄적인 유닛 테스트를 작성하도록 요청합니다. Claude가 놓친 엣지 케이스를 자동으로 포함해 일반적으로 상당한 정신적 에너지가 드는 작업을 몇 분 만에 완료합니다. 마치 검토할 수 있는 코딩 어시스턴트가 있는 것 같습니다.

**머신러닝 개념 설명**

머신러닝 배경 지식이 없는 팀원들은 모델 관련 함수와 설정을 설명받기 위해 Claude에 의존합니다. 구글 검색과 문서 읽기에 한 시간 걸리던 것이 10-20분으로 줄어들어 리서치 시간이 80% 감소했습니다.

**언어 간 코드 번역**

다른 프로그래밍 언어로 기능을 테스트할 때 테스트하고 싶은 것을 설명하면 Claude가 필요한 언어(예: Rust)로 로직을 작성합니다. 테스트 목적만으로 새 언어를 배울 필요가 없어졌습니다.

**명령어 기억 및 쿠버네티스 관리**

복잡한 쿠버네티스 명령어를 기억하는 대신 Claude에게 "모든 파드나 디플로이먼트 상태를 가져오는 방법" 같은 올바른 구문을 질문하면 인프라 작업에 필요한 정확한 명령어를 받습니다.

### 팀에 미친 영향

- **ML 개념 학습 가속화**: 리서치 시간이 80% 감소 - 구글 검색에 한 시간 걸리던 것이 10-20분입니다.
- **더 빠른 코드베이스 탐색**: 동료에게 묻는 대신 몇 초 만에 관련 파일을 찾고 시스템 아키텍처를 이해할 수 있습니다.
- **포괄적인 테스트 커버리지**: Claude가 엣지 케이스를 포함한 유닛 테스트를 자동 생성해 코드 품질을 유지하면서 정신적 부담을 덜어줍니다.
- **언어 장벽 제거**: Rust 같은 익숙하지 않은 언어로도 배우지 않고 기능을 구현할 수 있습니다.

### 추론 팀의 핵심 팁

1. **지식 베이스 기능 먼저 테스트**: 다양한 질문을 해보고 Claude가 구글 검색보다 빠르게 답변하는지 확인하세요. 더 빠르고 정확하다면 워크플로우에서 시간을 절약하는 귀중한 도구입니다.

2. **코드 생성부터 시작**: Claude에게 구체적인 지침을 주고 로직을 작성하도록 요청한 후 정확성을 검증하세요. 더 복잡한 작업에 사용하기 전에 도구의 역량에 대한 신뢰를 쌓을 수 있습니다.

3. **테스트 작성에 활용**: Claude가 유닛 테스트를 작성하게 하면 일상 개발 작업의 상당한 압박을 덜어줍니다. 이 기능을 활용해 모든 테스트 케이스를 수동으로 생각하지 않고도 코드 품질을 유지하세요.

## 데이터 사이언스 및 시각화 팀의 Claude Code 활용

데이터 사이언스 및 ML 엔지니어링 팀은 모델 성능을 이해하기 위해 정교한 시각화 도구가 필요하지만, 이러한 도구를 만들려면 종종 익숙하지 않은 언어와 프레임워크 전문 지식이 필요합니다. Claude Code는 이 팀들이 풀스택 개발자가 되지 않고도 프로덕션 품질의 분석 대시보드를 구축할 수 있게 합니다.

### 주요 Claude Code 활용 사례

**JavaScript/TypeScript 대시보드 앱 구축**

"JavaScript와 TypeScript를 거의 모르는" 상태에서 팀은 Claude Code로 RL 모델 성능과 학습 데이터를 시각화하는 전체 React 애플리케이션을 만듭니다. Claude에게 처음부터 전체 애플리케이션을 작성하도록 맡기는데, 코드를 직접 이해할 필요 없이 5,000줄짜리 TypeScript 앱 같은 것도 만듭니다. 시각화 앱은 상대적으로 컨텍스트가 적고 전체 모노레포를 이해할 필요가 없어서 학습과 평가 중 모델 성능을 이해하는 도구를 빠르게 프로토타이핑하는 데 중요합니다.

**반복적인 리팩토링 작업 처리**

편집기 매크로로는 너무 복잡하지만 대규모 개발 노력을 들일 정도는 아닌 머지 충돌이나 준복잡한 파일 리팩토링에 직면하면 Claude Code를 "슬롯머신"처럼 사용합니다 - 상태를 커밋하고 Claude가 30분간 자율적으로 작업하게 한 다음 솔루션을 수락하거나 작동하지 않으면 새로 시작합니다.

**일회성 노트북 대신 영구적 분석 도구 생성**

버려지는 일회성 Jupyter 노트북을 만드는 대신 팀은 Claude가 향후 모델 평가에서 재사용할 수 있는 영구적인 React 대시보드를 만들게 합니다. Claude의 성능을 이해하는 것이 "팀에게 가장 중요한 일 중 하나"이기 때문에 중요합니다 - 학습과 평가 중 모델이 어떻게 수행되는지 이해해야 하는데, "실제로 간단하지 않고 단순한 도구로는 단일 숫자가 오르는 것만 봐서는 많은 신호를 얻을 수 없습니다."

**의존성 없는 작업 위임**

완전히 익숙하지 않은 코드베이스나 언어의 작업은 전체 구현을 Claude Code에 위임해 모노레포에서 컨텍스트를 수집하고 실제 코딩 과정에 개입 없이 작업을 실행하는 능력을 활용합니다. 새 기술을 배우는 데 시간을 쓰는 대신 전문 분야 밖에서도 생산성을 발휘합니다.

### 팀에 미친 영향

- **2-4배 시간 절약 달성**: 지루하지만 수동으로 관리 가능했던 일상적인 리팩토링 작업이 훨씬 빠르게 완료됩니다.

2. **필요할 때 단순함을 위해 중단**: 감독하면서 Claude를 멈추고 "왜 이렇게 해? 더 간단한 걸 시도해봐"라고 주저 없이 요청하세요. 모델은 기본적으로 더 복잡한 솔루션을 선호하지만 더 간단한 접근 방식 요청에 잘 반응합니다.

## API 팀의 Claude Code 활용

API 지식 팀은 PDF 지원, 인용, 웹 검색 같은 추가 지식을 Claude의 컨텍스트 윈도우로 가져오는 기능을 담당합니다. 크고 복잡한 코드베이스에서 작업하면서 익숙하지 않은 코드 섹션을 끊임없이 마주치고, 주어진 작업에 어떤 파일을 검토해야 하는지 이해하는 데 상당한 시간을 쓰고, 변경하기 전에 컨텍스트를 쌓습니다. Claude Code는 시스템 아키텍처를 이해하고, 관련 파일을 식별하고, 복잡한 상호작용을 설명하는 가이드 역할을 해 이 경험을 개선합니다.

### 주요 Claude Code 활용 사례

**첫 단계 워크플로우 계획**

팀은 모든 작업의 "첫 번째 정거장"으로 Claude Code를 사용해 버그 수정, 기능 개발, 분석을 위해 어떤 파일을 검토해야 하는지 질문합니다. 코드베이스를 수동으로 탐색하고 작업 시작 전에 컨텍스트를 모으는 기존의 시간 소모적인 프로세스를 대체합니다.

**코드베이스 전반에서 독립적 디버깅**

팀은 다른 사람에게 도움을 요청하는 대신 코드베이스의 익숙하지 않은 부분에서 버그를 해결할 자신감이 생겼습니다. Claude에게 "이 버그 고칠 수 있을 것 같아? 이런 동작이 보이고 있어"라고 물으면 즉각적인 진전을 얻는 경우가 많은데, 이전에는 필요한 시간 투자 때문에 실행 가능하지 않았습니다.

**독푸딩을 통한 모델 반복 테스트**

Claude Code는 자동으로 최신 연구 모델 스냅샷을 사용하므로 모델 변경을 경험하는 주요 방법이 됩니다. 개발 주기 동안 모델 동작 변경에 대한 직접적인 피드백을 얻는데, 이전 출시 때는 경험하지 못했던 것입니다.

**컨텍스트 전환 오버헤드 제거**

코드 스니펫을 복사하고 문제를 광범위하게 설명하면서 파일을 Claude.ai로 드래그하는 대신 추가 컨텍스트 수집 없이 Claude Code에서 직접 질문할 수 있어 정신적 오버헤드가 크게 줄었습니다.

### 팀에 미친 영향

- **익숙하지 않은 영역 해결 자신감 증가**: 팀원들이 익숙하지 않은 코드베이스에서 독립적으로 버그를 디버그하고 인시던트를 조사할 수 있습니다.
- **컨텍스트 수집 시간 대폭 절약**: 코드 스니펫을 복사하고 파일을 Claude.ai로 드래그하는 오버헤드가 제거되어 정신적 컨텍스트 전환 부담이 줄었습니다.
- **더 빠른 로테이션 온보딩**: 새 팀으로 로테이션하는 엔지니어가 동료 상담 없이 빠르게 익숙하지 않은 코드베이스를 탐색하고 의미 있게 기여할 수 있습니다.
- **개발자 행복도 향상**: 일상 워크플로우의 마찰이 줄어들면서 팀이 더 행복하고 생산적으로 느낀다고 보고합니다.

### API 지식 팀의 핵심 팁

1. **한 번에 해결책이 아닌 협력적 파트너로 접근**: Claude가 즉시 문제를 해결할 것으로 기대하기보다 함께 반복하는 협력자로 접근하세요. 첫 시도에 완벽한 솔루션을 얻으려는 것보다 이 방식이 더 잘 작동합니다.

2. **익숙하지 않은 영역에서 자신감 구축에 활용**: 전문 분야 밖의 버그 해결이나 인시던트 조사를 주저하지 마세요 - Claude Code가 광범위한 컨텍스트 구축이 일반적으로 필요한 영역에서도 독립적으로 작업할 수 있게 합니다.

3. **최소한의 정보로 시작**: 필요한 것의 최소한만 가지고 시작하고 광범위한 설명을 미리 준비하기보다 Claude가 프로세스를 안내하게 하세요.

### 주요 Claude Code 활용 사례

**자동화된 Google Ads 크리에이티브 생성**

팀은 성과 지표가 포함된 수백 개의 기존 광고가 담긴 CSV 파일을 처리하고, 반복할 저성과 광고를 식별하고, 엄격한 글자 수 제한(헤드라인 30자, 설명 90자)을 충족하는 새 변형을 생성하는 에이전틱 워크플로우를 구축했습니다. 두 개의 전문 서브 에이전트(헤드라인용, 설명용)를 사용해 여러 캠페인에 걸쳐 수동 생성이 필요했던 것을 몇 분 만에 수백 개의 새 광고를 생성할 수 있습니다. 이전에는 상당한 시간이 걸렸을 대규모 테스트와 반복이 가능해졌습니다.

**대량 크리에이티브 제작을 위한 Figma 플러그인**

유료 소셜 광고용 정적 이미지를 수동으로 복제하고 편집하는 대신 프레임을 식별하고 헤드라인과 설명을 교체해 최대 100개의 광고 변형을 프로그래밍 방식으로 생성하는 Figma 플러그인을 개발했습니다. 몇 시간이 걸렸을 복사-붙여넣기 작업이 배치당 0.5초로 줄었습니다. 10배의 크리에이티브 출력이 가능해져 주요 소셜 채널에서 훨씬 더 많은 크리에이티브 변형을 테스트할 수 있습니다.

**캠페인 분석을 위한 Meta Ads MCP 서버**

Meta Ads API와 통합된 MCP 서버를 만들어 Claude Desktop 앱 내에서 캠페인 성과, 지출 데이터, 광고 효과를 직접 쿼리합니다. 성과 분석을 위해 플랫폼 간 전환이 필요 없어져 모든 효율성 향상이 더 나은 ROI로 이어지는 곳에서 중요한 시간을 절약합니다.

**메모리 시스템을 활용한 고급 프롬프트 엔지니어링**

광고 반복 전반에 걸쳐 가설과 실험을 기록하는 기초적인 메모리 시스템을 구현해 새 변형을 생성할 때 시스템이 이전 테스트 결과를 컨텍스트로 가져와 자기 개선하는 테스트 프레임워크를 만들었습니다. 수동으로 추적하기 불가능했을 체계적인 실험이 가능해졌습니다.

### 팀에 미친 영향

- **반복 작업에서 극적인 시간 절약**: 광고 카피 생성이 2시간에서 15분으로 줄어 전략적 업무에 시간을 확보했습니다.
- **크리에이티브 출력 10배 증가**: 자동 생성과 Figma 통합으로 채널 전반에서 훨씬 더 많은 광고 변형을 테스트할 수 있습니다.
- **더 큰 팀처럼 운영**: 전통적으로 전담 엔지니어링 리소스가 필요했던 작업을 처리할 수 있습니다.
- **전략적 초점 전환**: 수동 실행보다 전체 전략과 에이전틱 자동화 구축에 더 많은 시간을 쓸 수 있습니다.

### 그로스 마케팅 팀의 핵심 팁

1. **API 지원 반복 작업 식별**: API가 있는 도구(광고 플랫폼, 디자인 도구, 분석 플랫폼 등)를 사용하는 반복 작업이 포함된 워크플로우를 찾으세요. 자동화의 주요 후보이며 Claude Code가 가장 큰 가치를 제공하는 곳입니다.

2. **복잡한 워크플로우를 전문 서브 에이전트로 분리**: 하나의 프롬프트나 워크플로우에서 모든 것을 처리하려 하지 말고 특정 작업을 위한 별도의 에이전트를 만드세요(헤드라인 에이전트와 설명 에이전트처럼). 복잡한 요구사항을 다룰 때 디버깅이 쉬워지고 출력 품질이 향상됩니다.

3. **코딩 전에 철저히 브레인스토밍하고 프롬프트 계획**: Claude.ai로 전체 워크플로우를 생각하는 데 상당한 시간을 미리 투자한 다음 Claude.ai가 Claude Code가 참조할 포괄적인 프롬프트와 코드 구조를 만들게 하세요. 복잡한 작업에 압도당하지 않도록 한 번에 해결하기보다 단계별로 작업하세요.

## 제품 디자인 팀의 Claude Code 활용

제품 디자인 팀은 Claude Code, Claude.ai, Anthropic API를 지원하며 AI 제품 구축을 전문으로 합니다. 비개발자도 Claude Code로 디자인과 엔지니어링 사이의 전통적인 격차를 해소하고 엔지니어와의 광범위한 왕복 없이 디자인 비전을 직접 구현할 수 있습니다.

### 주요 Claude Code 활용 사례

**프론트엔드 폴리싱 및 상태 관리 변경**

시각적 조정(서체, 색상, 간격)을 위해 광범위한 디자인 문서를 만들고 엔지니어와 여러 차례 피드백을 거치는 대신 Claude Code로 이러한 변경을 직접 구현합니다. 엔지니어들은 그들이 "디자이너가 보통 하지 않는 대규모 상태 관리 변경"을 한다고 언급하며, 덕분에 정확히 구상한 품질을 달성할 수 있습니다.

**GitHub Actions 자동 티켓팅**

Claude Code의 GitHub 통합을 사용해 필요한 변경을 설명하는 이슈/티켓을 제출하기만 하면 Claude Code를 열지 않고도 Claude가 자동으로 코드 솔루션을 제안합니다. 폴리싱 작업 백로그에 대한 원활한 버그 수정 및 기능 다듬기 워크플로우가 만들어집니다.

**빠른 인터랙티브 프로토타이핑**

목업 이미지를 Claude Code에 붙여넣으면 엔지니어가 즉시 이해하고 반복할 수 있는 완전히 기능하는 프로토타입을 생성합니다. 광범위한 설명과 작동 코드로의 번역이 필요했던 정적 Figma 디자인의 전통적인 사이클을 대체합니다.

**엣지 케이스 발견 및 시스템 아키텍처 이해**

Claude Code로 오류 상태, 로직 흐름, 다양한 시스템 상태를 매핑해 개발 후가 아닌 디자인 단계에서 엣지 케이스를 식별합니다. 초기 디자인의 품질이 근본적으로 향상됩니다.

**복잡한 카피 변경 및 법적 준수**

전체 코드베이스에서 "리서치 프리뷰" 메시지를 제거하는 작업의 경우 Claude Code로 모든 인스턴스를 찾고, 주변 카피를 검토하고, 법무팀과 실시간으로 변경 사항을 조율하고, 업데이트를 구현했습니다 - 일주일의 왕복 조율 대신 30분짜리 통화 두 번으로 완료했습니다.

### 팀에 미친 영향

- **핵심 워크플로우 변화**: Claude Code가 주요 디자인 도구가 되어 Figma와 Claude Code를 80% 시간 동안 열어둡니다.
- **2-3배 빠른 실행**: 이전에 엔지니어와 광범위한 왕복이 필요했던 시각적 및 상태 관리 변경을 직접 구현합니다.
- **몇 주에서 몇 시간으로 사이클 타임 단축**: GA 출시 메시지 같은 복잡한 프로젝트가 일주일의 조율 대신 30분짜리 통화 두 번으로 완료됩니다.
- **두 가지 뚜렷한 사용자 경험**: 개발자는 "증강된 워크플로우"(더 빠른 실행)를, 비기술 사용자는 "이런, 나도 개발자야 워크플로우"(이전에 불가능했던 완전히 새로운 역량)를 얻습니다.
- **디자인-엔지니어링 협업 개선**: 디자이너가 시스템 제약과 가능성을 미리 이해해 더 나은 커뮤니케이션과 더 빠른 문제 해결이 가능합니다.

### 제품 디자인 팀의 핵심 팁

1. **엔지니어에게 적절한 설정 도움 받기**: 엔지니어링 팀원에게 초기 저장소 설정과 권한 도움을 받으세요 - 기술적 온보딩이 비개발자에게 어렵지만 일단 구성되면 일상 워크플로우에 변혁적입니다.

2. **커스텀 메모리 파일로 Claude 동작 안내**: 코딩 경험이 거의 없는 디자이너로서 상세한 설명과 더 작고 점진적인 변경이 필요하다고 Claude에게 알려주는 구체적인 지침을 만드세요. Claude 응답의 품질이 극적으로 향상되고 덜 위협적으로 느껴집니다.

3. **프로토타이핑에 이미지 붙여넣기 활용**: Command+V로 스크린샷을 Claude Code에 직접 붙여넣으세요 - 디자인을 읽고 기능하는 코드를 생성하는 데 탁월해 정적 목업을 엔지니어가 즉시 이해하고 발전시킬 수 있는 인터랙티브 프로토타입으로 바꾸는 데 매우 유용합니다.

## RL 엔지니어링 팀의 Claude Code 활용

RL 엔지니어링 팀은 강화학습의 효율적인 샘플링과 클러스터 전반의 가중치 전송에 집중합니다. 주로 소규모에서 중간 규모의 기능 작성, 디버깅, 복잡한 코드베이스 이해에 Claude Code를 사용하며 잦은 체크포인트와 롤백을 포함하는 반복적 접근 방식을 취합니다.

### 주요 Claude Code 활용 사례

**감독 하 자율성으로 기능 개발**

팀은 가중치 전송 컴포넌트의 인증 메커니즘 구현 같은 소규모에서 중간 규모 기능의 대부분 코드를 Claude Code가 작성하게 하면서 감독합니다. Claude가 주도하게 하되 잘못된 방향으로 갈 때 방향을 잡아주며 상호작용적으로 작업합니다.

**테스트 생성 및 코드 리뷰**

직접 변경을 구현한 후 Claude Code에 테스트를 추가하거나 코드를 리뷰하도록 요청합니다. 이 자동화된 테스트 워크플로우가 일상적이지만 중요한 품질 보증 작업에서 상당한 시간을 절약합니다.

**디버깅 및 오류 조사**

Claude Code로 오류를 디버깅하는데 결과가 엇갈립니다 - 때로는 문제를 즉시 식별하고 관련 테스트를 추가하지만 다른 때는 문제를 이해하는 데 어려움을 겪습니다. 그러나 전반적으로 작동할 때 가치를 제공합니다.

**코드베이스 이해 및 콜 스택 분석**

워크플로우에서 가장 큰 변화 중 하나는 Claude Code로 관련 컴포넌트와 콜 스택의 빠른 요약을 얻는 것으로, 수동 코드 읽기나 광범위한 디버깅 출력 생성을 대체합니다.

**쿠버네티스 운영 안내**

그렇지 않으면 광범위한 구글 검색이 필요할 쿠버네티스 운영에 대해 Claude Code에 자주 질문해 구성 및 배포 질문에 즉각적인 답변을 얻습니다.

### 개발 워크플로우 영향

- **실험적 접근 방식 가능**: "시도하고 롤백" 방법론을 사용해 체크포인트를 자주 커밋하므로 Claude의 자율 구현 시도를 테스트하고 필요하면 되돌릴 수 있어 더 실험적인 개발이 가능합니다.
- **문서화 가속화**: Claude Code가 상당한 시간을 절약하는 유용한 코멘트를 자동으로 추가하지만, 때로는 이상한 곳에 코멘트를 추가하거나 의문스러운 코드 구성을 사용한다고 언급합니다.
- **한계가 있는 속도 향상**: Claude Code가 "비교적 적은 시간"으로 소규모에서 중간 규모 PR을 구현할 수 있지만 첫 시도에 작동하는 것은 약 1/3 정도이며 추가 안내나 수동 개입이 필요합니다.

### RL 엔지니어링 팀의 핵심 팁

1. **특정 패턴을 위해 Claude.md 파일 커스터마이즈**: Claude.md 파일에 반복되는 도구 호출 실수를 방지하는 지침을 추가하세요. 예를 들어 "pytest를 실행하고 run하지 마, 불필요하게 cd하지 말고 올바른 경로를 사용해"라고 알려주면 일관성이 크게 향상됩니다.

2. **체크포인트 중심 워크플로우 사용**: Claude가 변경할 때 정기적으로 작업을 커밋해 실험이 작동하지 않을 때 쉽게 롤백할 수 있게 하세요. 위험 없이 더 실험적인 개발 접근이 가능합니다.

3. **먼저 한 번에 시도한 다음 협업**: Claude에게 빠른 프롬프트를 주고 전체 구현을 먼저 시도하게 하세요. 작동하면(약 1/3 확률) 상당한 시간을 절약합니다. 그렇지 않으면 더 협력적이고 안내된 접근으로 전환하세요.

## 법무 팀의 Claude Code 활용

법무 팀은 실험과 Anthropic 제품 제공에 대해 배우려는 욕구를 통해 Claude Code의 잠재력을 발견했습니다. 한 팀원은 가족을 위한 접근성 도구를 만들고 비개발자에게 기술의 힘을 보여주는 업무 프로토타입과 관련된 개인적인 활용 사례가 있었습니다.

### 주요 Claude Code 활용 사례

**가족을 위한 맞춤 접근성 솔루션**

팀원들은 의료 진단으로 인해 말하기 어려움이 있는 가족을 위한 커뮤니케이션 보조 도구를 만들었습니다. 단 한 시간 만에 네이티브 음성-텍스트 변환을 사용해 응답을 제안하고 음성 뱅크를 사용해 읽어주는 예측 텍스트 앱을 만들어 언어 치료사가 추천한 기존 접근성 도구의 공백을 해결했습니다.

**법무 부서 워크플로우 자동화**

팀원들이 Anthropic의 적합한 변호사와 연결할 수 있도록 도와주는 프로토타입 "전화 트리" 시스템을 만들어 법무 부서가 기존 개발 리소스 없이 일반적인 작업을 위한 맞춤 도구를 구축할 수 있음을 보여주었습니다.

**팀 조율 도구**

관리자들은 주간 팀 업데이트를 자동화하고 제품 전반의 법적 검토 상태를 추적하는 G Suite 애플리케이션을 만들어 변호사들이 스프레드시트 관리 대신 간단한 버튼 클릭으로 검토가 필요한 항목을 빠르게 표시할 수 있게 했습니다.

**솔루션 검증을 위한 빠른 프로토타이핑**

Claude Code로 도메인 전문가에게 보여줄 수 있는 기능적인 프로토타입을 빠르게 만들어(접근성 도구를 UCSF 전문가에게 보여주는 것처럼) 더 많은 시간을 투자하기 전에 아이디어를 검증하고 기존 솔루션을 식별합니다.

### 업무 스타일과 영향

- **Claude.ai에서 계획하고 Claude Code에서 구축**: Claude.ai에서 브레인스토밍하고 계획한 다음 Claude Code로 이동해 구현하는 2단계 프로세스를 사용하며, 한 번에 모든 것을 출력하기보다 속도를 늦추고 단계별로 작업하도록 요청합니다.
- **시각 우선 접근**: 스크린샷을 자주 사용해 Claude Code에 인터페이스가 어떻게 보이길 원하는지 보여주고 텍스트로 기능을 설명하기보다 시각적 피드백을 기반으로 반복합니다.
- **프로토타입 주도 혁신**: "바보 같은" 또는 "장난감" 프로토타입을 공유하는 것에 대한 두려움을 극복하는 것을 강조하는데, 이러한 시연이 다른 사람들에게 생각하지 못했던 가능성을 보게 합니다.

### 보안 및 규정 준수 인식

- **MCP 통합 우려**: 제품 변호사로서 깊은 MCP 통합의 보안 함의를 즉시 식별하며 AI 도구가 더 민감한 시스템에 접근함에 따라 보수적인 보안 태도가 장벽을 만들 것이라고 언급합니다.
- **규정 준수 도구 우선순위**: AI 역량이 확장됨에 따라 규정 준수 도구를 빠르게 구축하는 것을 옹호하며 혁신과 위험 관리 사이의 균형을 인식합니다.

### 법무 팀의 핵심 팁

1. **먼저 Claude.ai에서 광범위하게 계획**: Claude Code로 이동하기 전에 Claude의 대화형 인터페이스로 전체 아이디어를 구체화하세요. 그런 다음 Claude에게 구현을 위한 단계별 프롬프트로 모든 것을 요약하도록 요청하세요.

2. **점진적이고 시각적으로 작업**: Claude Code에게 속도를 늦추고 한 번에 한 단계씩 구현하도록 요청해 압도당하지 않고 복사-붙여넣기할 수 있게 하세요. 인터페이스가 어떻게 보이길 원하는지 보여주기 위해 스크린샷을 자유롭게 사용하세요.

3. **불완전해도 프로토타입 공유**: "장난감" 프로젝트나 미완성 작업을 숨기려는 충동을 극복하세요 - 프로토타입을 공유하면 다른 사람들이 가능성을 보게 되고 일반적으로 상호작용하지 않는 부서 간 혁신을 촉발합니다.

## 핵심 요약

- **데이터 인프라**: Claude.md 파일 문서화가 성능의 핵심, MCP 서버로 보안 통제, 팀 간 워크플로우 공유로 모범 사례 전파
- **제품 개발**: 자동 승인 모드로 빠른 프로토타이핑, 핵심 기능은 동기식 감독, 테스트 우선 생성으로 자기 충족적 루프 구축
- **보안 엔지니어링**: 커스텀 슬래시 명령어 적극 활용, Claude가 자율적으로 작업 후 주기적 확인, 문서화에도 활용
- **추론**: ML 개념 리서치 시간 80% 감소, 유닛 테스트 자동 생성으로 정신적 부담 경감
- **데이터 사이언스**: 슬롯머신 접근법(저장-실행-수락/재시작), 복잡한 솔루션 요청 시 단순화 요구
- **API**: 협력적 파트너로 접근, 익숙하지 않은 영역에서 자신감 구축, 최소 정보로 시작
- **그로스 마케팅**: API 지원 반복 작업 식별, 복잡한 워크플로우를 서브 에이전트로 분리, 사전 브레인스토밍 중요
- **제품 디자인**: 엔지니어 도움으로 초기 설정, 커스텀 메모리 파일로 동작 안내, 이미지 붙여넣기로 프로토타이핑
- **RL 엔지니어링**: Claude.md에 특정 패턴 추가, 체크포인트 중심 워크플로우, 한 번에 시도 후 협업
- **법무**: Claude.ai에서 계획 후 Claude Code에서 구축, 시각 우선 접근, 불완전해도 프로토타입 공유
