# Claude Code: 에이전틱 코딩 모범 사례

[원본 링크](https://www.anthropic.com/engineering/claude-code-best-practices)

**게시일:** 2025년 4월 18일

## 개요

이 종합 가이드는 Anthropic의 에이전틱 코딩 커맨드라인 도구인 Claude Code를 다양한 프로젝트와 환경에서 활용하는 검증된 기법을 다룹니다.

## 1. 설정 커스터마이징

### CLAUDE.md 파일

Claude가 자동으로 컨텍스트에 포함하는 특별한 설정 파일을 생성합니다. 다음 내용을 문서화해야 합니다:

- Bash 명령어
- 코드 스타일 가이드라인
- 테스트 지침
- 저장소 규칙
- 개발자 환경 설정

파일은 다음 위치에 배치할 수 있습니다:
- 프로젝트 루트
- 상위/하위 디렉토리
- 홈 폴더 (`~/.claude/CLAUDE.md`)

### 설정 튜닝

CLAUDE.md 파일을 반복적인 개선이 필요한 프롬프트처럼 다루세요. 효과를 테스트하고 지침을 개선합니다. `#` 키를 사용하면 Claude가 대화 컨텍스트를 기반으로 이 파일을 자동 업데이트합니다.

### 도구 권한

허용된 도구 목록을 다음을 통해 관리합니다:
- "항상 허용" 프롬프트
- `/permissions` 명령어
- JSON 수동 편집
- `--allowedTools` 같은 CLI 플래그

## 2. 도구 접근 확장

### Bash 도구

CLAUDE.md 파일에 사용 예시와 함께 커스텀 도구를 문서화합니다.

### MCP 서버

다음을 통해 연결합니다:
- 프로젝트 설정
- 전역 설정
- `.mcp.json` 파일

### 커스텀 슬래시 명령어

매개변수를 위한 `$ARGUMENTS`와 함께 템플릿을 `.claude/commands/` 폴더에 저장합니다.

## 3. 일반적인 워크플로우

### 탐색, 계획, 코딩, 커밋

이 다용도 패턴은 다음을 포함합니다:
1. 먼저 관련 파일 읽기
2. 상세 계획 작성 (확장된 사고를 위해 "think" 사용)
3. 솔루션 구현
4. 변경사항 커밋

### 테스트 주도 개발

1. 예상 입력/출력을 기반으로 먼저 테스트 작성
2. 테스트 실패 확인
3. 테스트 커밋
4. 테스트를 통과하도록 코드를 반복적으로 구현

### 시각적 반복

1. 디자인 목업이나 스크린샷 제공
2. Claude가 코드 구현
3. 스크린샷을 찍어 비교
4. 결과가 일치할 때까지 반복

### 안전한 YOLO 모드

격리된 컨테이너에서 `--dangerously-skip-permissions`를 사용하여 다음과 같은 작업을 중단 없이 수행합니다:
- 린팅 수정
- 보일러플레이트 생성

## 4. 최적화 기법

### 구체성의 중요성

상세한 지침은 첫 시도 성공률을 크게 향상시킵니다.

### 시각적 커뮤니케이션

다음을 활용합니다:
- 스크린샷
- 드래그 앤 드롭 이미지
- 디자인 참조 및 디버깅을 위한 파일 경로

### 방향 수정

- 계획 단계 활용
- Escape를 눌러 중단
- Escape 두 번 탭하여 이전 방향으로 돌아가기
- `/clear` 명령어로 컨텍스트 집중

### 체크리스트와 스크래치패드

복잡한 마이그레이션이나 다수의 수정 작업에는 마크다운 파일을 작업 체크리스트로 사용합니다.

### 데이터 입력 방법

- 콘텐츠 복사-붙여넣기
- stdin을 통한 데이터 파이프
- Claude에게 도구를 통해 데이터를 가져오도록 지시
- 파일 경로와 URL 제공

## 5. 헤드리스 모드 자동화

다음을 위해 프롬프트와 함께 `-p` 플래그를 사용합니다:
- CI/CD
- pre-commit 훅
- 자동화된 워크플로우

### 핵심 패턴

**팬아웃:** 작업 목록을 생성하고 항목을 프로그래밍 방식으로 처리

**파이프라이닝:** `--output-format stream-json`으로 Claude를 기존 데이터 처리 워크플로우에 통합

## 6. 멀티 Claude 워크플로우

### 관심사 분리

한 Claude가 코드를 작성하고 다른 Claude가 리뷰하거나, 독립적인 작업에 별도 인스턴스를 사용합니다.

### Git 워크트리

`git worktree` 명령어를 사용하여 머지 충돌 없이 서로 다른 브랜치에서 동시에 세션을 실행합니다.

### 프로그래밍 방식 통합

다음을 위해 커스텀 하네스와 함께 헤드리스 모드를 사용합니다:
- 대규모 코드 변환
- 데이터 처리

## 실용적 활용 사례

이 가이드에는 다음과 같은 예시가 포함되어 있습니다:

- **코드베이스 Q&A 및 온보딩** - 새로운 코드베이스를 빠르게 이해
- **Git 히스토리 탐색** - 커밋 히스토리 분석 및 커밋 메시지 생성
- **GitHub 상호작용** - PR, 이슈 분류, 코드 리뷰 자동화
- **Jupyter 노트북 개선** - 노트북 개선 및 문서화
- **이슈 분류 자동화** - 이슈 자동 분류 및 우선순위 지정
- **주관적 코드 리뷰** - 전통적인 린팅을 넘어서

## 결론

이 패턴들은 엄격한 요구사항이 아닌 커뮤니티에서 검증된 접근 방식입니다. 개별 워크플로우에 맞게 실험하고 커스터마이징할 때 최상의 결과를 얻을 수 있습니다.
