# 좋은 컨텍스트가 좋은 코드를 만든다: StockApp이 AI 네이티브 엔지니어링 문화를 구축한 방법

[원본 링크](https://blog.stockapp.com/good-context-good-code/)

**저자:** Justin M Berman (Waleed Kadous, Charles Feng, Dennis Yilmaz, Amr Elsayed, Mohammed Mogasbe, James Feng, Bruno Fantauzzi 기여)

**날짜:** 2025년 8월 5일 | 9분 분량

---

## 개요

StockApp 엔지니어링 팀은 기존 워크플로우에 AI를 끼워 맞추는 대신, 처음부터 AI 협업을 중심으로 개발 프로세스를 설계했다. 이들의 연구에 따르면 이 접근법은 수동 개발 대비 약 2.5배, 부분적으로 AI를 도입한 기존 문화 대비 약 2배의 생산성 향상을 보였다.

핵심 원칙은 다음과 같다: 인간과 AI 에이전트가 체계적으로 컨텍스트를 구축하고 공유할 때 우수한 소프트웨어가 탄생한다.

---

## 다섯 가지 핵심 원칙

### 1. 공유 작업 공간으로서의 모노레포

저장소는 인간과 기계 모두를 위한 것이다. 기존 코드베이스와 달리 StockApp은 문서를 코드와 동일한 수준으로 엄격하게 다룬다. 주요 구성 요소는 다음과 같다:

- **docs/designs/**: 제품 요구사항과 스키마 ("왜"와 "무엇")
- **docs/plans/**: 구현 단계, 주로 인간과 에이전트가 함께 작성 ("어떻게")
- **docs/guides/**: API 및 도구 튜토리얼
- **schema.sql**: 데이터 구조의 단일 진실 공급원
- **README.md & CLAUDE.md**: 코드베이스 전반에 걸친 지역화된 지침

팀은 "자연어가 프로그래밍 언어만큼 중요하다"고 강조한다.

### 2. 점진적 컨텍스트를 활용한 계층적 개발

개발은 하향식 접근법을 따른다:

1. 설계 단계—인간이 요구사항 제공, 에이전트가 문서 초안 작성, 팀이 반복 개선
2. 계획—에이전트가 설계를 단계별 작업으로 변환하여 인간이 검토
3. 구현—인간의 감독 하에 에이전트가 대부분의 코딩 수행
4. 안전장치—테스트와 보호 장치로 컨텍스트 손실 방지
5. 리뷰—원래 설계 목표에 대한 최종 검증
6. 정제—향후 참조를 위해 문서와 스키마 업데이트

### 3. 에이전트의 전면적 활용

팀은 수동 대안을 고려하기 전에 거의 모든 작업에 에이전트를 활용한다. 독특한 활용 사례는 다음과 같다:

- **커밋 및 PR 메시지 작성**
- **코드 변경 후 문서 업데이트**
- **CLAUDE.md 지침 작성** (에이전트가 인간보다 더 나은 자기 지시를 제공)
- **테스트 케이스 생성** (과도한 모킹 방지)
- **공동 디버깅**—인간이 근본 원인을 가정하고 에이전트가 검증
- **코드 품질 개선**—중복/사용하지 않는 코드 및 보안 이슈 식별
- **Git 및 병합 충돌 해결**

기사에서 언급하듯이, "이러한 수준의 에이전트 자율성은 마법이 아니라 공유 컨텍스트를 생성하고 유지하기 위한 체계적인 투자의 직접적인 결과다."

### 4. MCP 서버와 커맨드라인 통합

팀은 약 6개의 Model Context Protocol 서버를 활용한다:

- **Notion과 Linear**: 기능 설명 접근 및 프로젝트 상태 업데이트
- **AWS와 SQL 데이터베이스**: 직접 로그 분석 및 데이터 검증
- **Git과 GitHub**: 히스토리 코드 조사 및 PR 관리

특히 Zen MCP 서버는 Claude Code가 Gemini나 o3 같은 경쟁 모델에게 피드백을 요청할 수 있게 하여 모델 간 비교를 가능하게 한다.

### 5. 앙상블 접근법이 개별 에이전트를 능가한다

머신러닝 원칙(random forest, bagging, boosting)에서 영감을 얻어, 팀은 인간 승인 전 리뷰 사이클에 여러 에이전트를 투입한다. 각 모델은 서로 다른 강점을 보인다—Gemini는 보안 이슈 식별에 뛰어나고, o3는 프로덕션 관점의 시각을 제공한다.

앙상블에는 인간도 포함되어, 어떤 단일 참여자보다 우수한 협업적 의사결정을 만들어낸다.

## 핵심 인사이트

기사는 효과적인 AI 네이티브 개발이 더 적은 것이 아니라 *더 많은* 엔지니어링 전문성을 요구한다고 강조한다. 컨텍스트를 정의하려면 어떤 정보가 중요하고 에이전트가 이를 어떻게 해석하는지 신중하게 고려해야 하는데, 이는 지식이 내부에만 있을 때는 없던 과제다. 또한 에이전트 실패는 상당한 파급 범위를 가질 수 있어, 자율적 운영보다는 경험 많은 인력의 세심한 감독이 필요하다.
