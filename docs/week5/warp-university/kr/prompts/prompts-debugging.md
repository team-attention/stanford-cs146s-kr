# 앱 디버깅을 위한 더 나은 프롬프트

[영상 바로가기](https://youtu.be/YzZmrusN8Cw)

## 핵심 요약

- REST API 개발의 오버헤드를 줄이기 위한 효과적인 프롬프트 전략
- 유닛 테스트와 보안 테스트를 명시적으로 요청하여 품질 향상
- 생성된 테스트를 rules 파일에 추가하면 AI가 코드 품질을 자체 검증
- Warp가 테스트를 참조 기준으로 사용하여 구현 가치를 판단

---

## 전체 번역

프로젝트를 위한 REST API를 만드는 것은 많은 오버헤드를 동반합니다. 테스트 스위트가 이러한 문제를 완화하는 데 도움이 되지만, 기존 방식대로 하면 평범한 결과만 얻게 됩니다. 여기, 여러분이 사용할 수 있는 더 나은 프롬프트가 있습니다. 이 프롬프트는 AI와 함께 앱을 개발하는 속도를 훨씬 빠르게 만들어줍니다. 이 프롬프트에서는 우리가 찾고자 하는 것을 명확하게 명시합니다. 유닛 테스트와 보안 테스트입니다. 이를 통해 변경사항을 커밋하거나 배포할 준비가 되면 간단한 명령어로 실행할 수 있는 테스트가 생성됩니다. 하지만 더 중요한 것은, 이 명령어를 rules 파일에 추가할 수 있다는 점입니다. 그러면 Warp가 이러한 테스트를 참조 기준(source of truth)으로 사용하여, Warp가 작성한 코드가 실제로 구현할 가치가 있는지 판단하게 됩니다.
