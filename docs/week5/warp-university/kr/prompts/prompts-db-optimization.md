# 데이터베이스 최적화를 위한 우선순위 매트릭스 만들기

[영상 바로가기](https://youtu.be/VgE5wgtDSnk)

## 핵심 요약

- **문제점**: 데이터베이스는 앱 성능의 주요 병목 지점이며, "최적화"는 다양한 의미를 가질 수 있음
- **솔루션**: 포괄적인 프롬프트로 코드베이스 전체의 SQL 사용처를 분석
- **분석 방법**: SQL의 내장 EXPLAIN 메서드를 활용하여 성능 데이터 수집
- **주요 검사 항목**: N+1 문제, 누락된 인덱스 등 일반적인 성능 저하 원인
- **결과물**: 영향도, 위험도, 노력도를 기준으로 한 우선순위 매트릭스
- **활용**: 데이터베이스 최적화 작업의 우선순위를 효과적으로 결정

## 전체 번역

앱 성능에서 가장 큰 병목 지점 중 하나는 바로 데이터베이스입니다. 그리고 데이터베이스가 속도를 느리게 만드는 방법은 매우 다양합니다.

그래서 AI에게 "이 데이터베이스 쿼리를 최적화해줘"라고 요청하면, 말 그대로 수백만 가지 의미가 될 수 있습니다. 더 안전하게, 더 빠르게, 읽기 쉽게 등등 말이죠. 그래서 여기 Warp에서 사용할 수 있는 프롬프트가 있는데, 이 프롬프트는 쿼리를 모든 측면에서 분석해줍니다. 이 프롬프트는 매우 길지만, 여러분에게 엄청난 시간을 절약해줄 것입니다. 이 프롬프트는 전체 코드베이스를 살펴보고 SQL이 사용되는 모든 영역을 찾아냅니다. 우리는 Warp에게 SQL의 내장 explain 메서드를 사용하여 소스로부터 직접 성능 정보를 가져오도록 요청합니다. 그런 다음 N+1 문제, 누락된 인덱스 등과 같이 속도 저하를 일으키는 일반적인 시나리오들을 검토합니다. 저는 여기에 우선순위 매트릭스를 추가했는데, 이 매트릭스는 Warp가 발견한 모든 최적화 사항을 보여주고 영향도, 위험도, 노력도를 기준으로 그래프를 작성합니다. 데이터베이스는 전체 웹 스택의 심장과도 같습니다. 그러니 이 유용한 프롬프트를 놓치지 마세요.
