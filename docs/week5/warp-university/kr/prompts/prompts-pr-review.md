# 시니어 개발자처럼 PR 리뷰하기

[영상 바로가기](https://youtu.be/NVwqQyphlAw)

## 전체 번역

대규모 PR을 리뷰하는 일은 때때로 까다로울 수 있습니다. 텍스트 덩어리가 빽빽하게 들어차 있기도 하죠. 이렇게 AI에게 요약을 요청할 수도 있습니다. "이 PR을 요약해줘." 하지만 문제는 전체 그림을 파악하지 못하고 AI에게 검증을 맡기게 된다는 점입니다. 이는 매우 위험할 수 있습니다. 다음은 PR 리뷰를 훨씬 빠르게 만들어주는 프롬프트입니다. 이 프롬프트는 요약 대신 인덱스와 우선순위 목록을 만들어줍니다. 이를 통해 대규모 PR을 효과적으로 탐색할 수 있는 경로를 얻을 수 있습니다. 또한 AI가 잡아낼 수 있는 명백한 위험 신호도 식별해줍니다. 약한 모델을 사용하는 경우, 구조화된 출력을 활용하여 제공된 출력을 보장받을 수 있습니다. AI는 매우 유용한 도구이지만, 애플리케이션에 들어가는 작업은 항상 직접 확인해야 합니다.

---

## 핵심 요약

- 대규모 PR 리뷰 시 단순 요약은 위험 - 전체 그림을 놓칠 수 있음
- AI에게 인덱스와 우선순위 목록을 요청하여 효과적인 리뷰 경로 확보
- AI가 명백한 위험 신호를 먼저 잡아내게 하여 리뷰 효율 향상
- 구조화된 출력으로 약한 모델에서도 일관된 결과 보장
- AI는 보조 도구일 뿐, 최종 검증은 개발자가 직접 수행해야 함
