# 규칙으로 코딩 모범 사례 설정하기

[영상 바로가기](https://youtu.be/AuM2OIvXlnY)

## 전체 번역

코드를 작성할 때 규칙을 사용하는 것이 좋습니다.

이는 꽤 세부적이고 개인적인 내용이 될 수 있지만, 우선 에이전트에게 코드 형식을 지정하고 작업을 확인하도록 지시하는 것부터 시작하는 것이 좋습니다. 또한 TypeScript에서 인터페이스보다 타입을 사용하는 것과 같은 선호사항을 설명하세요. 이렇게 하면 에이전트가 처음부터 코드를 올바르게 작성하고 솔루션을 반환하기 전에 오류를 잡아낼 수 있습니다. 저는 또한 어니스트 헤밍웨이의 간결한 글쓰기 스타일에서 영감을 받은 헤밍웨이 테스트를 사용하여 코드 주석 품질을 개선하는 것을 좋아합니다. 이렇게 하면 JS docs를 읽기 좋고 깔끔하게 유지할 수 있습니다.

---

## 핵심 요약

- **코드 작성 규칙**: 에이전트에게 코드 형식 지정과 작업 검증을 지시
- **개인 선호사항 명시**: TypeScript에서 타입 vs 인터페이스 등 구체적 선호사항 전달
- **오류 사전 방지**: 솔루션 반환 전에 에이전트가 스스로 오류 검출
- **헤밍웨이 테스트**: 간결하고 읽기 쉬운 코드 주석 작성을 위한 원칙
- **JS docs 품질**: 명확하고 읽기 좋은 문서화 유지
