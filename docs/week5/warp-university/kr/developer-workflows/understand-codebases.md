# 코드베이스 이해하기

[영상 바로가기](https://youtu.be/11rz9OYQ8Hg)

## 핵심 요약

- Warp의 AI 기능을 사용하여 익숙하지 않은 코드베이스를 빠르게 이해할 수 있습니다
- 시맨틱 검색과 GPT 기반 검색을 결합하여 관련 코드를 정확하게 찾습니다
- 백만 줄 이상의 대규모 코드베이스에서도 효과적으로 작동합니다
- 코드 구조, 데이터 플로우, 컴포넌트 관계를 자동으로 분석하고 설명합니다
- 큰 파일은 자동으로 분할하여 관련 섹션만 읽어 효율성을 높입니다

---

## 전체 번역

저는 Warp를 사용해서 익숙하지 않은 코드베이스 부분을 어떻게 학습하는지 보여드리겠습니다. Warp의 자체 코드베이스에서, 최근에 추가한 새로운 에이전트 팝업으로 시연하겠습니다. 이 팝업에 버튼을 추가하고 싶은데, 어떻게 동작하는지 모르고 코드의 어느 부분에 있는지도 모릅니다. 그래서 Warp에게 설명을 요청하겠습니다.

"에이전트 팝업 코드가 어떻게 구조화되어 있는지, 코드베이스 어디에 있는지, 그리고 어떻게 렌더링되고 호출되는지 설명해주세요. 새로운 에이전트 버튼을 추가하는 방법을 알아낼 수 있도록 해당 컴포넌트의 전체 데이터 플로우와 구조를 이해하고 싶습니다."

실행되는 동안 몇 가지 멋진 기능을 설명하겠습니다. 말씀드렸듯이 Warp의 코드베이스는 백만 줄이 넘는 Rust 코드에 커스텀 UI 프레임워크를 사용하고 있습니다. 따라서 이해하기 간단한 것이 아닙니다. 먼저 시맨틱 검색을 사용하는데, 벡터 임베딩 검색을 통해 적절한 코드가 어디에 있을지 파악합니다. 잘 찾았네요. 제가 생각한 파일인 agent management popup을 찾았습니다. 그런 다음 GPT를 사용하여 코드베이스 전체에서 특정 문자열을 검색하는 모드로 전환합니다. 서로 다른 종류의 검색을 번갈아 가며 수행하는 것을 볼 수 있습니다. 여기서는 특정 파일을 읽고 있는데, 부끄럽게도 이 파일은 만 줄짜리 파일입니다. 그래서 더 작은 조각으로 나누고 관련 섹션만 읽는데, 이는 현명한 방법입니다.

그런 다음 반복적으로 이러한 심볼들을 검색하고 파일을 읽습니다. 잠시 동안 이 작업을 수행한 후 구조에 대한 좋은 설명을 제공할 수 있게 됩니다. 결과를 살펴보겠습니다. 맞는 것 같네요. 여기가 팝업이 정의된 곳입니다. 여기가 워크스페이스에 렌더링되는 곳이고, 이것들이 관련된 액션들입니다. 각 코드 조각에 대한 개요를 제공해 줍니다. 와, 정말 멋지네요. 탭바가 어떻게 렌더링되고 팝업이 어디에 추가되는지 보여주고 있습니다. 매우 상세한 설명을 해주고 있는데, 정확히 제가 원하던 것입니다. 그리고 제가 설명한 기능을 구축하기 위해 어떻게 시작할 수 있는지 알려줍니다. 감사합니다. 도움이 되셨기를 바랍니다.
