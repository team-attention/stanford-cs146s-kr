# Warp에서 Figma Remote MCP 설정하기

[영상 바로가기](https://youtu.be/PsM_Y8Pt-1Q)

## 전체 번역

우리는 방금 Warp에서 실행되는 원격 Figma MCP 서버에 대한 지원을 출시했습니다.

원격 MCP 서버는 다른 곳에 존재하며 Warp를 통해 연결할 수 있습니다.

이것은 모든 프로세스를 직접 관리할 필요가 없다는 것을 의미합니다. 제가 해야 할 일은 이 간단한 JSON을 붙여넣는 것뿐이고, Warp가 OAuth 로그인 플로우를 열어줍니다. Figma 계정으로 한 번만 로그인하면 끝입니다. 이것은 설정 시간을 절약하고, 토큰 관리의 번거로움을 제거하며, 시작하는 것을 훨씬 쉽게 만들어줍니다.

이제 Figma MCP 서버는 AI 코딩 도구가 Figma 파일에서 직접 컨텍스트를 가져올 수 있게 하여 LLM이 프로젝트에서 환각을 일으킬 가능성을 줄입니다. 따라서 AI가 관계를 볼 수 있도록 상위 레벨 컨텍스트를 위한 스크린샷을 가져올 수 있습니다. 컴포넌트, 변수, 스타일과 같은 디자인 시스템 규칙을 통해 생성된 코드가 디자인 시스템과 일치하도록 합니다. Figma 파일에서 코드를 반환하는 인코드를 가져오고, 텍스트, 이미지, 레이어 이름을 포함하는 콘텐츠 메타데이터를 가져와 AI가 현실적인 데이터 모델을 채울 수 있도록 돕습니다.

여기서 저는 이 Figma 파일에서 웹사이트를 만들기 위해 Warp에 프롬프트를 붙여넣고 있습니다. 디자인 세부사항, 기술 요구사항, 구체적인 지침과 같이 원하는 모습에 대한 기본 가이드라인을 추가하고 있습니다. Figma 파일을 참조하려면 마우스 오른쪽 버튼을 클릭하고, 복사, 다른 이름으로 붙여넣기를 선택한 다음 선택 항목에 대한 링크 복사를 클릭하기만 하면 됩니다.

그러면 Warp는 Figma MCP 서버를 호출하여 메타데이터를 가져오고, 스크린샷을 찍고, 코드를 가져올 수 있습니다. 그런 다음 많은 코드 diff를 생성하기 시작하고 5분 이내에 이 웹사이트를 로컬에서 실행할 수 있습니다. 분명히 완벽하지는 않지만 Figma 디자인의 구조를 꽤 잘 모방합니다. 많은 로고와 이미지가 누락되어 있습니다. 그래서 Warp는 제가 만든 assets 폴더에 다운로드하라고 지시했습니다. 그래서 그렇게 한 다음, 그 이미지들로 웹사이트를 채우라고 말하는 두 번째 반복을 시작하겠습니다.

이것이 실행되는 동안, Warp의 지속적인 입력 기능이 얼마나 멋진지 강조하고 싶었습니다. 저는 Misho 로고를 업로드하는 것을 잊었고 Warp가 플레이스홀더로 자체 로고를 만들려고 하는 것을 알아차렸습니다. 저는 빠르게 assets 폴더에 이미지를 업로드하고 Warp에 알렸고, Warp는 프로세스 중간에 업로드된 로고를 찾아 사용하도록 조정할 수 있었습니다.

그리고 여기 완성된 결과가 있습니다. 다시 말하지만, 여전히 다듬어야 할 부분이 많지만, Figma 원격 MCP 서버 덕분에 20분 이내의 작업치고는 꽤 좋습니다.

요약하자면, Warp는 이제 Figma MCP 서버를 원격으로 실행하는 것을 지원하며, OAuth 지원을 추가하여 설정을 극적으로 단순화했습니다. 이는 Figma뿐만 아니라 Sentry, GitHub, Linear와 같은 모든 즐겨찾는 MCP 서버를 더 쉽게 설정할 수 있다는 것을 의미합니다. 도움이 되기를 바랍니다.

## 핵심 요약

- Warp가 원격 Figma MCP 서버 지원을 출시하여 로컬 프로세스 관리가 불필요함
- OAuth 로그인으로 간단한 JSON 붙여넣기만으로 설정 완료 (토큰 관리 불필요)
- Figma MCP 서버는 AI 도구가 Figma 파일에서 직접 컨텍스트를 가져와 LLM 환각 감소
- 스크린샷, 디자인 시스템 규칙, 코드, 메타데이터를 AI에 제공하여 정확한 코드 생성
- Figma 파일 링크를 복사하여 Warp 프롬프트에 붙여넣기만 하면 웹사이트 생성 가능
- Warp의 지속적인 입력 기능으로 작업 중간에도 피드백 반영 가능
- Sentry, GitHub, Linear 등 다른 MCP 서버도 동일한 방식으로 쉽게 설정 가능
