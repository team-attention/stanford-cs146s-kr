# UI 변경하기

[영상 바로가기](https://youtu.be/V2pwBN6Vt7k)

## 전체 자막

좋습니다. Warp의 코딩 기능을 사용해서 Warp에 기능을 구현하는 방법을 보여드리겠습니다. 꽤 작은 변경이지만, 아이디어는 충분히 전달될 것입니다. 우리가 하려는 일은 무엇일까요? 열린 Linear 이슈가 하나 있습니다. 스파클 아이콘을 변경하려고 합니다. 이 아이콘이 바로 여기 있는 것인데요. 이것 대신에 에이전트 아이콘을 사용하려고 합니다. 그럼 에이전트에게 프롬프트를 주고 작업을 시작하게 하겠습니다. 어떻게 진행되는지 봅시다.

첨부된 Linear URL 형식에 따라 새 브랜치를 생성해주세요. 참고로, 저는 방금 Warp에 말로 명령했습니다. 특히 이런 데모에서는 타이핑을 많이 하는 것보다 말하는 게 더 쉽다고 생각합니다. 또한 제가 미리 만들어둔 이 아이콘들이 어떻게 생겼는지 보여주는 스크린샷도 첨부했습니다. 그래야 어떤 건지 알 수 있으니까요. 에이전트 모드와 스파클 아이콘이 어떻게 생겼는지에 대한 스크린샷도 첨부했습니다. 그 아이콘들을 이해하고, 코드에서 사용된 곳을 찾아서, 스파클을 사용하는 모든 곳을 에이전트 모드로 바꿔주세요. 특히 히스토리 메뉴에서 이 작업을 확실히 해주세요. 하지만 다른 곳에서도 했으면 좋겠습니다. 실제로 코딩 변경을 시작하기 전에 계획을 먼저 알려주세요. 꽤 상세한 프롬프트를 주고 시작하겠습니다. 어떻게 하는지 봅시다. 맞는 것 같네요.

이렇게 하죠. 그리고 네, 함수 이름 변경을 고려하세요. "네, 더 이상 스파클이 아니니까 이름을 바꿔야 합니다"라고 말하겠습니다. 그래요, 진행합시다. 그리고 함수에 더 나은 이름을 지어주세요. render AI sparkles icon 대신에 render the agent mode icon이나 render agent mode icon 같은 식으로요. 변경을 해주세요. 그리고 어떻게 됐는지 알려주세요.

이게 진행되는 동안, 사실 이게 멋있다고 생각합니다. diff가 생성되는 걸 볼 수 있습니다. Warp를 diff를 자동으로 수락하는 모드로 설정했습니다. diff가 무엇인지 확인하고 싶으면, 바로 여기서 볼 수 있습니다. 실제 렌더링 코드를 변경하고 있고, 제가 원하는 대로 함수 이름도 변경하고 있습니다. 예를 들어 이런 변경사항들이 자동으로 적용되는 걸 원하지 않는다면, 여기 AI 설정으로 가서 어떻게 적용될지 제어할 수 있습니다. 저는 이 시점에서 AI를 정말 신뢰하기 때문에 이것들을 매우 관대하게 설정해뒀습니다. git 샌드박스에서 작업하고 있으니까요. 이제 계속 진행하면서 다른 모든 인스턴스를 찾을 것입니다.

진행되는 동안 무엇을 하고 있는지 볼 수 있습니다.

좋습니다. 이제 변경사항을 만들었고, 어떻게 컴파일하는지도 충분히 알고 있습니다. cargo check를 실행하고 있네요. 진행 상황을 보고 싶으면, 그냥 들어가서 이렇게 하면 됩니다.

좋습니다. 뭔가 놓친 것 같네요.

음, 이런 일이 일어나죠. 사람도 때로는 import를 추가하는 걸 잊습니다. 그럼 이걸 고칠 수 있는지 봅시다. 좋습니다, 이제 컴파일됐습니다. presubmit 스크립트를 실행하려고 하네요. 그건 하지 않고, 대신 그냥 실행해서 실제로 변경이 됐는지 봅시다. 이게 진행되는 동안, 이게 백만 줄이 넘는 Rust 코드로 이루어진 코드베이스에서 작업하고 있다는 걸 말씀드리고 싶습니다. 완전히 맞춤형 UX 프레임워크를 사용하고요. 그래서 에이전트가 하기에 가장 쉬운 작업은 아닙니다. 이런 종류의 일을 할 수 있다는 게 매우 인상적입니다.

좋습니다, 여기 있네요. 에이전트 아이콘이요.

꽤 멋지네요. 좋아 보인다고 말하고 싶습니다.

바로 여기요. 요약하자면, 에이전트 모드가 Linear 티켓을 이해하게 했습니다. 기능을 구축하기 위해 몇 가지 이미지를 제공했습니다. 해달라고 요청했죠. 계획을 세웠고, 브랜치를 만들었고, 변경사항을 만들었습니다. 올바르게 컴파일될 때까지 반복했죠. Pull request를 만드는 것까지 계속할 수도 있지만, 여기서 멈추는 게 좋은 지점이라고 생각합니다.

## 핵심 요약

- **실제 개발 워크플로우**: Warp의 AI 에이전트를 사용하여 UI 아이콘 변경 작업을 수행하는 실제 사례
- **음성 기반 프롬프트**: 타이핑 대신 음성으로 명령하고 스크린샷을 첨부하여 맥락 제공
- **계획 후 실행**: 에이전트가 코드 변경 전에 계획을 먼저 제시하도록 요청
- **자동 diff 적용**: 실시간으로 코드 변경사항을 확인하고 자동으로 적용하는 모드 활용
- **반복적 수정**: 컴파일 에러 발생 시 에이전트가 자동으로 수정을 반복
- **대규모 코드베이스**: 백만 줄 이상의 Rust 코드와 맞춤형 UX 프레임워크에서도 효과적으로 작동
- **완전한 워크플로우**: Linear 티켓 이해 → 브랜치 생성 → 코드 변경 → 컴파일 확인까지 자동화
