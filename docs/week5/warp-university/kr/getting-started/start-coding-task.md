# Warp로 코딩 작업 시작하기

[영상 바로가기](https://youtu.be/IuFSuOYstfg)

## 핵심 요약

- Warp를 사용해 Warp의 버그를 고치는 실제 워크플로우 시연
- 음성 입력 기능으로 긴 프롬프트를 빠르게 작성 가능
- 이미지 첨부, 파일 참조, URL 추가 등 다양한 컨텍스트 제공 방법
- Warp는 코드베이스 자동 인덱싱으로 관련 파일을 스마트하게 검색
- 실용적 예시: "프롬프트에 모델 선택기 표시" 설정 버그 수정

## 전체 번역

Warp를 사용해 코딩하는 방법, 특히 Warp를 사용해서 Warp의 버그를 고치는 과정을 보여드리겠습니다. 약간 메타적인 작업이죠. 여기 이 버그가 있는 Warp 개발자 빌드가 있습니다. 보시다시피 Warp에는 모델 선택기가 있습니다. 그리고 이 버전의 Warp에서 적용되지 않는 설정이 하나 있는데, 바로 "프롬프트에 모델 선택기 표시" 설정입니다. 기억하기 위해 스크린샷을 찍어두겠습니다. 그런 다음 Warp를 사용해서 이 버그를 어떻게 고치는지 보여드리겠습니다. 이제 개발자 빌드를 종료하고, Warp에게 버그를 고쳐달라고 프롬프트를 입력하겠습니다. 음성으로 할 건데요, 긴 프롬프트를 타이핑하는 것보다 훨씬 빠르거든요. Warp에 음성으로 말하고 싶으시면 여기 마이크 아이콘을 클릭하시면 됩니다. 처음 클릭하고 나면 단축키를 설정할 수 있는데, 저는 그렇게 사용하고 있습니다. "유니버설 개발자 입력이 프롬프트에 모델 선택기 표시 설정을 적용하지 않는 버그가 있습니다. 이 버그를 고쳐주시고, 설정 변경사항을 리스닝하도록 해서 누군가 체크박스를 클릭하거나 해제하면 개발자 입력이 적절하게 업데이트되도록 해주세요. 이 수정을 할 때 첨부된 스크린샷과 참조 파일을 확인해주세요." 이렇게 Warp에 말을 하고, Warp가 더 잘 작동하도록 컨텍스트를 제공하는 여러 방법이 있습니다. 하나는 이미지를 첨부하는 것입니다. 방금 찍은 스크린샷을 첨부하겠습니다.

두 번째는 파일이나 심볼을 참조하는 것입니다. 여기서는 관련이 있다고 생각하는 "유니버설 개발자 입력"이라는 파일을 참조하겠습니다.

그건 그렇고, 여기에 원시 URL을 바로 붙여넣을 수도 있습니다. 일반적으로 파일을 참조할 필요가 없는 이유는 Warp가 코드베이스를 인덱싱하고 올바른 파일을 찾기 위한 다양한 검색 도구를 가지고 있기 때문입니다.
