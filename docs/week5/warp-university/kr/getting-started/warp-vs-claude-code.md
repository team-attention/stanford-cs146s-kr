# Warp vs Claude Code 비교

[영상 바로가기](https://youtu.be/NUVftxAqZQo)

## 전체 자막

안녕하세요, Warp의 제품 엔지니어 Ben Holmes입니다. Claude Code와 Warp의 내장 코딩 에이전트를 빠르게 비교해보려고 합니다.

먼저 Claude Code에 대해 이야기해보겠습니다.

Claude는 CLI 도구이므로 Warp처럼 터미널에서 시작해야 하고, Claude CLI를 설치해야 합니다. 그런 다음 Claude를 실행하여 코딩 에이전트에 프롬프트를 입력할 수 있습니다. 이 박스에 프롬프트를 입력하면 Claude Code가 작업을 수행합니다. 파일을 읽고, 코드베이스에서 핵심 용어를 검색할 수 있으며, 충분한 컨텍스트를 확보하면 사용자를 대신하여 diff를 생성할 수 있습니다. 여기서는 readme 파일을 편집하고 있습니다. 수동으로 검토할 수 있는 diff를 제시하거나, shift tab을 눌러 나머지 세션 동안 편집을 자동으로 승인할 수 있습니다. 또한 계획 모드로 들어가서 에이전트가 작업을 시작하기 전에 조사를 하고 마크다운 기반 계획을 제시하도록 할 수 있습니다. Warp의 코딩 에이전트는 Warp에 직접 내장되어 있습니다. 따라서 이미 터미널로 Warp를 사용하고 있다면 AI 쿼리를 제출할 수 있습니다. AI 쿼리처럼 보이면 에이전트 모드로 자동 전환되며, 키보드 단축키나 여기 있는 작은 에이전트 버튼을 사용하여 언제든지 수동으로 입력할 수 있습니다. 쿼리를 제출하면 Warp 코딩 작업이 시작됩니다. Claude Code와 유사하게 파일을 읽고, 코드베이스를 검색하고, 코드 diff를 생성할 수 있습니다. 이제 한 가지 주요 차이점은 이러한 diff로 무엇을 할 수 있느냐입니다. 세션의 나머지 부분에 대해 승인하거나 자동 승인할 수 있지만, 해당 diff를 직접 편집할 수도 있습니다. diff를 수동으로 편집하거나, 에이전트가 추가했을 수 있는 주석을 제거하는 등의 작업을 위해 내장 편집기와 같은 편리한 기능을 제공합니다. 승인을 누르면 에이전트가 세션 후반에 덮어쓰려고 시도하지 않도록 해당 diff에 대해 알립니다.

또 다른 주요 차이점은 변경 사항을 검토하는 것입니다. Claude Code는 diff를 검토하려면 git CLI나 VS Code 같은 도구를 사용해야 하지만, Warp에는 이를 위한 내장 버튼이 있습니다. 세션이 끝날 때 "변경사항 검토"를 누르거나 여기 있는 이 버튼을 누르면 에이전트가 작업한 모든 변경 사항을 볼 수 있는 코드 diff 뷰가 나타납니다. 다단계 PR 작업 중이고 해당 세션에서 작업한 모든 변경 사항을 보고 싶다면 메인 라인과 비교할 수도 있습니다. 계획에 관해서는 Warp에는 쿼리가 충분히 집약적으로 들리면 에이전트가 계획을 요청할 수 있도록 하는 옵션이 있습니다. 여기 이 쿼리에서는 계획을 요청하기로 결정했습니다. 계획 없이 진행할 수도 있고, Claude Opus를 사용하여 조사를 하고 Claude Code와 유사한 마크다운 계획을 제시할 수도 있습니다. 이제 컨텍스트 수집에 대해 이야기해보겠습니다. Claude Code에서는 코드베이스를 탐색하고 어떤 컨텍스트가 관련될 수 있는지 파악할 수 있지만, @ 기호를 사용하여 명시적인 컨텍스트를 추가할 수 있습니다. 이렇게 하면 현재 디렉토리의 모든 파일에 대한 파일 선택기가 제공됩니다. 따라서 이를 참조하고 직접 질문할 수 있습니다. Warp의 코딩 에이전트도 명시적으로 컨텍스트를 참조할 수 있습니다. @ 기호를 누르면 현재 프로젝트의 파일을 참조할 수 있습니다. 하지만 심볼도 참조할 수 있습니다. 이것은 매우 강력합니다. 참조하고 싶은 특정 함수나 객체가 있으면 해당 함수의 이름, 속한 파일, 정확한 줄 번호를 인라인으로 제공하여 더 구체적인 질문을 할 수 있습니다. 파일을 직접 찾고 싶다면 프로젝트 탐색기를 사용할 수 있습니다. 이것은 프로젝트의 모든 파일을 볼 수 있는 간단한 파일 트리를 불러옵니다. 파일을 클릭하면 인라인 편집기가 열려 파일을 직접 편집하거나 LLM 쿼리의 컨텍스트를 위해 해당 파일의 특정 부분을 참조할 수 있습니다. 이렇게 하면 에이전트가 어디를 봐야 하는지 알 수 있도록 줄 번호와 파일 이름도 삽입됩니다.

이제 모델 선택에 대해 논의하겠습니다.

Claude는 슬래시 메뉴에서 모델을 포함한 여러 옵션을 사용할 수 있습니다.

이를 통해 Claude의 모델 중에서 선택할 수 있습니다. 따라서 Warp에서 조사 작업을 위한 Claude Sonnet, Opus 및 Opus 계획 모드가 있습니다.

모델 드롭다운을 눌러 여기서 모델을 선택할 수 있습니다. 이를 통해 Claude 및 Claude Opus와 함께 GPT-5의 다양한 추론 수준을 가진 Gemini 및 GPT-5 시리즈 모델을 포함한 다양한 모델에 액세스할 수 있습니다. 이제 구성 옵션에 대해 논의하겠습니다. Claude Code에서는 슬래시 명령에서 도구의 모든 측면을 구성할 수 있습니다. 이를 통해 현재 상태, 사용할 모델을 변경하는 모델 전환기, 허용되거나 거부된 도구를 선택하는 권한 메뉴, 특정 코딩 작업을 위한 서브 에이전트를 생성하는 에이전트 생성기와 같은 간단한 옵션을 볼 수 있습니다. Warp로 이동하면 설정을 구성하는 몇 가지 다른 방법이 있습니다. 여기서도 슬래시 명령을 사용하여 MCP 도구, 프롬프트 및 글로벌 규칙을 추가할 수 있습니다. Claude Code는 특정 git 리포지토리에 규칙 범위를 지정하지만 Warp에서는 모든 프로젝트가 어떻게 작동해야 하는지 설명하는 글로벌 규칙 디렉토리를 가질 수 있으며 여기에서 액세스할 수 있습니다. Warp는 또한 파일 검색을 개선하기 위해 코드베이스에 대한 임베딩을 생성하기 위해 여기서 시작할 수 있는 코드베이스 인덱싱을 제공합니다. 에이전트 구성의 경우 프로필을 사용하여 편집할 수 있습니다. 여기 Warp 설정 메뉴에 있고 기본 계획 및 읽기 및 쓰기 권한에 대한 기본 설정으로 기본 프로필을 설정했습니다. 또한 수동 검토 없이 항상 코드 diff를 허용하는 조금 더 완화된 다른 프로필도 만들었습니다.

이러한 도구를 비교해보면, 둘 다 에이전트가 수행할 수 있는 작업에 대한 광범위한 구성을 제공하지만 서브 에이전트와 후크도 허용하는 Claude Code에 약간의 우위를 주고 싶습니다. 이제 시간이 지남에 따라 에이전트를 관리하는 것에 대해 논의하겠습니다. Claude Code는 CLI 도구이므로 해당 CLI 탭을 확인하여 에이전트가 코딩 작업에서 차단되었는지 확인해야 합니다. 하지만 에이전트가 현재 수행 중인 작업을 보기 위해 탭 설명을 업데이트합니다. Warp 코딩 에이전트는 도구에 내장되어 있으므로 에이전트가 프로세스의 어느 위치에 있는지 표시하는 상태 표시기, 도구 팁의 더 설명적인 쿼리, 에이전트가 차단될 때마다 알림 토스트와 같이 에이전트가 수행 중인 작업에 대한 더 많은 정보를 제공할 수 있으므로 해당 탭으로 빠르게 이동하여 조치를 취할 수 있습니다. 또한 원하는 경우 소리와 함께 또는 소리 없이 데스크톱 알림을 받도록 선택할 수 있습니다.

이제 성능을 비교하기 위해 인코딩 작업을 실행해 보겠습니다. 왼쪽에 Warp가 있고 오른쪽에 Claude Code가 있습니다. 그리고 각각에게 Sentry 이슈를 제공하겠습니다. URL을 전달하겠습니다. 그리고 그 이상의 지침은 제공하지 않겠습니다. 이 작업을 각 코딩 에이전트에서 세 번씩 실행하여 코드 품질과 소요 시간에 대한 성능을 확인했습니다. 여기서 각 에이전트가 Claude Sonnet을 사용하여 2분에서 4분 사이의 시간이 걸리며 평균 시간에서 Claude Code에 약간의 우위가 있음을 볼 수 있습니다. 코딩 전 조사의 품질을 살펴보면 둘 사이에 유사한 성능을 보입니다. 각각은 Sentry MCP 서버를 식별할 수 있었지만, Claude Code 시험에서 fetch 쿼리를 사용한 예가 하나 있었습니다. 하지만 MCP 서버를 명시적으로 사용하도록 요청하면 다시 정상 궤도로 돌아왔습니다. 또한 이슈의 근본 원인을 식별할 수 있었는데, 이것은 render keyboard shortcut 함수입니다. 이것은 사용자가 설정에서 바인딩을 해제할 수 있었을 때 항상 키보드 단축키가 바인딩되어 있을 것으로 예상하는 유틸리티입니다. Claude Code의 코드 출력을 먼저 살펴보면 솔루션의 품질이 혼합되어 있음을 알 수 있습니다.

여기서는 모두 작동하지만 키 입력을 렌더링하려고 시도할 때 사용할 수 없는 경우 일부 빈 UI를 스텁하고 있음을 볼 수 있습니다. 하지만 이 render 함수를 전혀 호출하지 않는 것이 더 나았을 것입니다. 호출하기 전에 여기에 검사를 추가하여 절반 정도 식별할 수 있었지만 아래에 불필요한 추가 검사도 추가했습니다. Warp는 모두 컴파일되고 실행되는 세 가지 다른 솔루션을 제공했습니다. 이 예에서는 render 함수에서 선택적 타입을 반환하기로 결정하여 결과를 사용할지 여부를 결정할 수 있습니다. 이것은 아마도 모든 시험 중에서 제가 가장 좋아하는 예이지만, 제가 직접 작성했다면 if 검사를 넣었을 것입니다. 또한 이 margin 값으로 빈 flex row를 렌더링하기로 결정한 Warp 측의 한 가지 이상값을 주목하세요. 어디서 왔는지도 모르겠지만, Claude는 항상 예측 가능한 것은 아닙니다. 그리고 약간의 와일드 카드로 Warp 코딩 에이전트에서 GPT-5도 사용하기로 결정했습니다.

이것은 분명히 Claude Code로는 할 수 없는 것입니다. 따라서 직접적인 비교가 아닐 수 있지만 Warp Code에서는 여러 모델을 사용할 수 있습니다.

따라서 GPT나 Gemini를 시도해서 코드베이스에서 무엇이 작동하는지 확인하는 것이 좋습니다. 그리고 낮은 추론 모델을 사용한 이번 시험에서 정말 훌륭한 결과를 얻을 수 있었습니다. GPT-5가 높은 품질의 출력으로 매우 일관된 1분 12초에서 20초의 성능을 발휘하는 것을 여기서 볼 수 있습니다. 모든 솔루션이 컴파일되고 문제없이 실행되었습니다. 그리고 매번 매우 유사한 답변을 제공했습니다. 앞서 언급한 것처럼 제가 했을 솔루션은 아니지만 빈 UI를 스텁하고 있지만 간결하고 작동하는 솔루션이었습니다. 따라서 결론을 내릴 시간입니다. Warp Code 또는 Claude Code? Claude 모델 세트를 사용하고 싶고 UI 대신 터미널 환경에서 작업하는 것을 선호한다면 Claude Code를 선택할 수 있습니다. 하지만 컨텍스트를 위한 파일 트리, 코드 diff, GPT-5와 같은 다양한 모델 선택과 같은 Warp의 코딩 기능을 중요시한다면 Warp Code가 좋은 선택이 될 것입니다.

## 핵심 요약

- **도구 특징**: Claude Code는 CLI 도구로 터미널에서 실행되며, Warp는 터미널에 내장된 코딩 에이전트를 제공
- **Diff 편집**: Warp는 에이전트가 생성한 diff를 내장 편집기로 직접 수정할 수 있으며, 변경사항을 에이전트에게 알림
- **변경사항 검토**: Claude Code는 git CLI나 VS Code 필요, Warp는 내장 "변경사항 검토" 버튼 제공
- **컨텍스트 참조**: 둘 다 @ 기호로 파일 참조 가능, Warp는 추가로 심볼(함수, 객체) 참조 및 프로젝트 탐색기 제공
- **모델 선택**: Claude Code는 Claude 모델만, Warp는 Claude, GPT-5, Gemini 등 다양한 모델 지원
- **구성 옵션**: 둘 다 광범위한 구성 제공, Claude Code는 서브 에이전트와 후크 지원으로 우위
- **에이전트 관리**: Warp는 상태 표시기, 토스트 알림, 데스크톱 알림으로 에이전트 상태를 더 명확하게 표시
- **성능 비교**: Sentry 이슈 해결 테스트 결과, Claude Sonnet 사용 시 유사한 성능, GPT-5는 일관된 고품질 출력 제공
- **선택 기준**: Claude 모델과 터미널 환경 선호 시 Claude Code, 파일 트리, 코드 diff, 다양한 모델 선택 등 코딩 기능 중시 시 Warp Code 추천
