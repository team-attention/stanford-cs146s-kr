# AI가 작성한 코드 인라인 편집

[영상 바로가기](https://youtu.be/dm-P63USsVg)

## 전체 번역

이제 이 에이전트를 시작하고 실행합니다. 제가 제공한 프롬프트 정보를 사용할 것입니다.

여기에 작업 목록을 만들고, 모든 파일을 살펴보면서 이 수정을 어떻게 할지 파악하려고 합니다. 진행 상황을 보여주고, 작업들을 표시하며, 우리는 그냥 작업하는 것을 지켜보면 됩니다.

앞서 언급했듯이, 다양한 검색 도구를 사용합니다. 여기서는 grep을 사용하고 있습니다. 또한 "searching codebase"라고 나올 때는 코드베이스 임베딩을 사용하여 검색을 수행합니다. 좋습니다, 버그를 이해했고 이제 diff를 생성하기 시작할 것이라고 알려줍니다.

Warp가 다른 터미널 기반 코딩 솔루션에 비해 독보적으로 강력한 이유 중 하나는 바로 이 diff입니다. 원한다면 그대로 승인할 수도 있고, 개선하기 위한 몇 가지 작업을 할 수도 있습니다. 하나는 refine을 클릭하거나 command R을 눌러서 AI에 다시 프롬프트를 보낼 수 있습니다.

그러면 프롬프트를 사용하여 diff를 다시 생성하도록 요청합니다. 또한 직접 편집할 수도 있습니다. Warp에는 내장 코드 에디터가 있어서, 이 edit 버튼을 클릭하면 실제 에디터 뷰로 들어갑니다. 굳이 편집할 필요는 없습니다. 이 diff는 올바릅니다. apply changes를 누르면 Warp가 이 수정을 완료하는 데 필요한 나머지 변경사항을 계속 생성할 것입니다.

이것도 제가 보기에 올바른 것 같습니다.

그래서 이 변경사항을 적용하겠습니다. 이제 여기 빨리 감기를 클릭해서 Warp가 그냥 실행되도록 하겠습니다. 모든 diff를 수동으로 승인하지는 않을 것입니다. Warp의 또 다른 장점은 언제 확인이 필요한지 자율성 수준을 제어할 수 있다는 것입니다. 에이전트별로 할 수도 있고, 설정으로 들어가서 보다 일반적으로 구성할 수도 있습니다. 설정으로 가려면 command P를 누르고, open settings를 선택한 다음, AI로 이동하면 에이전트가 가진 자율성 권한을 선택할 수 있습니다. 다시 돌아가겠습니다. 이제 버그를 수정했고 자동으로 컴파일 명령을 실행하고 있습니다. 컴파일이 확인되었습니다. 다음으로 포맷하려고 합니다. control C를 눌러 중지하고, 이제 코드를 실행해서 수정이 완료되었는지 확인하겠습니다. 이게 Rust의 특징입니다. 컴파일하는 데 시간이 좀 걸립니다. 좋습니다, 이제 실행 중입니다.

버그가 수정되었는지 확인해봅시다. 이 버그는 이 체크박스가 작동하지 않는 것이었습니다. 이제 여기를 보면 모델 선택기가 사라졌고, 이것이 우리가 원하던 것입니다. 이것을 클릭하면 모델 선택기가 다시 나타납니다. Warp가 순전히 프롬프트만으로 버그를 수정했습니다. 정확히 무엇을 하고 있는지 볼 수 있었습니다. Warp가 코드 편집을 지원하기 때문에 편집할 기회도 있었습니다. 올바른 파일들을 모두 찾았고, 빌드도 해주었으며, git 상태도 확인해주었습니다. 이 작업을 수행하는 것이 훨씬 더 쉬워졌습니다.

## 핵심 요약

- Warp 에이전트는 프롬프트를 기반으로 자동으로 버그를 수정하고 작업 목록을 생성
- diff 뷰에서 변경사항을 미리 확인하고, 승인하거나 refine 또는 직접 편집 가능
- 내장 코드 에디터를 통해 AI가 생성한 코드를 인라인으로 수정
- 에이전트별 또는 전역 설정을 통해 자율성 수준 제어 가능
- grep, 코드베이스 임베딩 등 다양한 검색 도구를 활용하여 관련 파일 탐색
- 변경사항 적용 후 자동으로 컴파일 및 검증 수행
- 프롬프트만으로 버그를 완전히 수정하고 빌드 및 git 상태까지 관리
