# 프로젝트 규칙 사용하기

[영상 바로가기](https://youtu.be/SCYovBn4TnM)

## 전체 번역

Warp의 프로젝트 규칙 파일을 사용하면 환각 없이 대규모 프로젝트를 훨씬 쉽게 구축하고 LLM이 프로젝트에 집중하도록 할 수 있습니다. 매번 설정을 다시 설명하는 대신, 한 번만 문서화하면 Warp가 항상 그 컨텍스트를 유지합니다. 이 영상에서는 Warp 마크다운 파일이 무엇인지 설명하고, 예시를 통해 살펴본 다음, 이를 최대한 활용하기 위한 모범 사례를 공유하겠습니다.

시작은 간단합니다. 먼저 `/init`을 실행하면 Warp가 프로젝트 루트에 `warp.md`라는 시작 파일을 생성합니다. 이제 실제 예시를 살펴보겠습니다. 저는 Firebase와 Vercel을 사용하는 Astro와 React 프로젝트를 위한 파일을 만들었습니다.

프로젝트 루트에서 `ls`를 입력하면 `warp.md` 파일이 바로 거기에 있는 것을 볼 수 있습니다. `/open project rules`를 실행하면 Warp가 사이드 파일 에디터에서 파일을 열어주고, 스크롤하면서 내부 내용을 모두 볼 수 있습니다. 마크다운 파일이 표준 마크다운 헤딩을 사용하여 파일을 논리적 섹션으로 구성하는 등의 모범 사례를 따르는 것을 볼 수 있습니다.

첫 번째 섹션은 일반적인 명령어를 다룹니다. 개발 서버를 시작하는 `npm run dev`, 프로덕션 빌드를 위한 `npm run build`, 의존성 설치를 위한 `npm install` 같은 것들입니다. 이를 통해 Warp가 프로젝트를 실행하고 빌드하는 재현 가능한 방법을 얻습니다.

다음은 아키텍처 개요입니다. 높은 수준에서 보면, 이 앱은 개발자가 brew 파일을 업로드하고, Firebase에 저장하고, 성격 분석을 실행하고, 결과를 웹 UI에 표시하도록 합니다. 파일에는 시스템을 쉽게 이해할 수 있도록 플로우 다이어그램과 컴포넌트 상호작용 다이어그램도 포함되어 있습니다.

거기서부터 프론트엔드는 Astro를 사용하고, 백엔드는 서버리스 API 라우트를 사용하고, 데이터베이스는 Firebase이며, 배포는 Vercel을 사용한다는 등의 주요 아키텍처 세부사항을 다룹니다. 또한 컴포넌트, 타입, 유틸리티, Firebase 설정을 어디서 찾을 수 있는지와 같은 프로젝트 구조도 문서화합니다.

이제 프로젝트 규칙을 최대한 활용하기 위한 몇 가지 팁을 공유하고 싶습니다. 첫 번째는 매우 간결하고 의도적으로 작성하는 것입니다. `warp.md` 파일의 모든 내용이 프롬프트 앞에 추가됩니다. 즉, 파일이 길수록 토큰 예산을 소비합니다. 사용되는 토큰이 많을수록 각 요청이 계산적으로 더 비싸집니다. 따라서 제 조언은 간결하게 유지하고 정말 중요한 것만 포함하라는 것입니다.

두 번째는 보일러플레이트로 시작하지만 이 파일을 살아있는 문서로 취급하라는 것입니다. Warp의 `/init` 명령은 견고한 기준선을 제공하지만, 진정한 가치는 반복에서 나옵니다. 예를 들어, 항상 main에서 브랜치를 만들고, 설명적인 이름을 사용하고, 초기에 드래프트 PR을 열고, 병합할 때만 스쿼시하는 것과 같은 팀의 Git 워크플로우를 추가할 수 있습니다. 프로젝트가 성장함에 따라 Warp가 여러분처럼 코드를 작성하도록 만드는 선호사항을 계속 입력해야 합니다.

세 번째는 하위 디렉토리에 규칙 파일을 생성하는 것입니다. 대규모 모노레포에서 작업하는 경우, 모든 것을 하나의 문서에 넣을 필요가 없습니다. 예를 들어, `source`로 이동한 다음 다시 `/init`을 실행하여 해당 디렉토리에만 맞춤화된 `warp.md`를 생성할 수 있습니다. 물론, 이 프로젝트는 너무 작아서 여기서는 약간 중복되지만, 요점은 이해하셨을 것입니다.

마지막으로, 시간이 지남에 따라 규칙 파일을 최적화해야 합니다. 특히 파일이 500줄 이상과 같이 방대해지면 `warp.md` 파일을 프롬프트 옵티마이저에 통과시킬 수 있습니다. 이렇게 하면 중복을 찾아내고, 겹치는 부분을 제거하고, 파일을 슬림화하여 추가 토큰을 소모하지 않으면서도 효과적으로 유지할 수 있습니다.

요약하자면, `warp.md` 파일은 프로젝트의 AI 온보딩 가이드입니다. 간결하게 유지하고, 진행하면서 업데이트하고, 시간이 지남에 따라 최적화하세요. 도움이 되었기를 바랍니다.

## 핵심 요약

- **프로젝트 규칙 파일**: Warp의 `warp.md` 파일은 LLM에게 프로젝트 컨텍스트를 제공하여 환각을 줄이고 일관성을 유지
- **시작 방법**: `/init` 명령으로 프로젝트 루트에 `warp.md` 생성
- **파일 구성**: 일반 명령어, 아키텍처 개요, 프로젝트 구조를 표준 마크다운으로 문서화
- **모범 사례**:
  - 간결하게 유지 (토큰 예산 관리)
  - 살아있는 문서로 관리 (팀 워크플로우, 선호사항 추가)
  - 하위 디렉토리별 규칙 파일 생성 (모노레포)
  - 프롬프트 옵티마이저로 최적화 (500줄 이상 시)
