# Warp 전체 개요

[영상 바로가기](https://youtu.be/O5E6ze3vqeo)

## 핵심 요약

- **Warp는 에이전틱 개발 환경**: 에이전트에게 프롬프트를 주는 것이 주요 상호작용 방식
- **하이브리드 인터페이스**: 전통적인 터미널 명령어와 자연어 프롬프트 모두 지원
- **자연어 명령**: 영어로 입력하면 에이전트가 자동으로 작업 수행
- **컨텍스트 인식 에이전트**: 요청을 처리하기 위해 필요한 명령어 실행 및 정보 수집 자동화

## 전체 번역

안녕하세요, Warp의 창립자이자 CEO인 Zach입니다.

오늘 Warp를 여러분께 소개하게 되어 기쁩니다. Warp를 열면 커맨드 라인 인터페이스를 보게 됩니다. 이것은 에이전틱 개발 환경(agentic development environment)으로, 주요 상호작용 방식이 에이전트에게 무언가를 하도록 프롬프트를 주는 것입니다. 하지만 Warp에서는 일반적인 커맨드 라인처럼 터미널 명령어를 실행할 수도 있습니다. 여기서 ls나 pwd를 실행하면 터미널 명령어가 실행되는 것을 볼 수 있습니다. Warp에서 영어로 입력하면, 에이전트를 실행하는 것으로 해석되어 에이전트가 여러분을 위해 무언가를 수행합니다.

예를 들어, "describe my open git changes"라고 입력하면 에이전트 플로우가 시작되고, 에이전트는 기본적으로 쿼리에 답하거나 요청한 작업을 수행하는 데 필요한 명령어를 실행하거나 컨텍스트를 수집합니다. 이 예시는 정말 간단합니다. 현재 git 브랜치에 대해 알려주고 있습니다.
