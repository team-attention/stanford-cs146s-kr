# 에이전트 프로필 사용하기

[영상 바로가기](https://youtu.be/iD0R-8fY-tY)

## 전체 자막

오늘은 Warp의 에이전트 프로필에 대해 알려드리겠습니다. 에이전트 프로필을 사용하면 다양한 상황에서 코딩 에이전트가 어떻게 동작할지 설정할 수 있습니다. 이 영상에서는 두 가지 다른 에이전트 프로필이 동일한 코딩 프로젝트를 어떻게 처리하는지 보여드리겠습니다. 프로젝트는 NFL 예측 앱을 만드는 것입니다. 아이디어는 간단합니다. 지난 10년간의 NFL 데이터를 스크래핑하고, 이를 점수, 팀, 선수 같은 구조화된 통계로 처리한 다음, 그 데이터를 사용해 다가오는 시즌의 승리를 예측하는 것입니다. 프롬프트에는 데이터 소스, 기술적 제약사항, 의존성, 정확한 CLI 명령어, 구현 세부사항, 최종 결과물에 대한 지침이 포함되어 있습니다.

먼저 전략적 에이전트입니다. 기본 모델로는 GPT5를 사용했습니다. GPT5는 효율적으로 코드를 생성하면서도 강력한 추론 능력을 제공하기 때문입니다. 계획 모델로는 Claude 4 Opus를 선택했습니다. Opus는 복잡한 프로젝트를 길고 상세한 단계별 계획으로 나누는 데 뛰어나기 때문입니다.

전략적 에이전트는 코드 diff 적용과 파일 읽기는 에이전트가 결정하도록 설정되어 있습니다. 계획 생성은 항상 허용되고, 명령어 실행은 항상 물어보도록 되어 있습니다. 이 에이전트로 NFL 예측 프롬프트를 실행하면, 가장 먼저 하는 일은 명확화 질문을 하는 것입니다. 선수 통계와 일정을 모두 스크래핑할까요, 아니면 하나에 먼저 집중할까요? 원시 데이터 파일을 로컬 폴더에 저장할까요, 아니면 데이터베이스에 저장할까요? 에이전트가 계속 진행하기 전에 이러한 질문에 답변해 달라고 요청합니다.

그런 다음 에이전트는 14단계 계획을 생성합니다.

매우 철저하게 프로젝트 구조 설정, 의존성 구성, Pydantic을 사용한 타입 안전성 추가, 수집 모듈 구축, 검증 레이어 생성 등을 포함합니다. 이제 중간쯤 왔을 때 NFL 일정 URL을 설정하라고 요청하며 일시 정지합니다.

기본적으로 에이전트는 "경기 일정을 어디서 다운로드해야 하나요?"라고 묻는 것입니다. 그리고 세 가지 지원되는 옵션을 제공했습니다.

각 시즌의 일정 파일을 동적으로 링크를 만들 수 있도록 환경 변수를 설정하기를 원했습니다. 저는 무슨 일이 일어나는지 그렇게 면밀히 따라가고 있지 않았습니다. 그래서 옵션 B, CSV 버전을 선택했습니다. 하지만 생성된 URL들은 계속해서 404 에러를 반환했습니다. 실제 NFLverse 저장소가 구조를 변경했기 때문입니다. 계획은 탄탄했지만, 일정 데이터를 가져올 수 없어서 실행이 멈춰버렸습니다.

다음으로 YOLO 에이전트로 바꿔봤습니다. YOLO 에이전트는 코드 diff 적용과 파일 읽기가 항상 허용되도록 설정되어 있습니다.

계획 생성은 절대 안 하고, 명령어 실행은 항상 허용됩니다. 보시다시피 이 에이전트 프로필은 훨씬 더 적은 가드레일을 가지고 있습니다. 허가를 요청하지 않고 긴 계획으로 귀찮게 하지도 않습니다. 13단계 대신 더 짧은 10단계 계획을 만들었습니다. 상세한 검증 모듈, 오케스트레이터, 광범위한 테스트 스캐폴딩 같은 것들을 건너뛰었습니다. 대신 프로젝트 초기화, CLI 구현, NFLverse 선수 데이터 수집, 점수 계산 구축, 데이터 변환 등 필수 요소로 바로 갔습니다. 그리고 중요한 것은, 일정부터 시작하는 대신 안정적이고 신뢰할 수 있는 URL을 가진 선수 통계로 바로 갔다는 것입니다.

기본적으로 YOLO 에이전트는 첫 번째 시도에서 일정 데이터를 전혀 가져오지 않기로 결정했습니다. 대신 주간 선수별 통계, 즉 패싱, 러싱, 리시빙 수치 같은 것들을 중심으로 핵심 파이프라인을 구축했습니다. 이러한 엔드포인트는 일관되고 변경되지 않기 때문입니다.

일정을 피함으로써, 전략적 에이전트에게 404 에러를 일으키던 취약한 파이프라인 부분을 건너뛰었습니다. 최종 결과는 YOLO 에이전트가 선수 성과에 대한 작동하는 데이터셋을 생성했고 심지어 최종 요약 테이블까지 만들었다는 것입니다.

여기서 트레이드오프를 명확하게 볼 수 있습니다. 전략적 에이전트는 체계적이고, 투명하며, 설정 가능합니다. 입력을 요청하고, 안전망을 구축하며, 무슨 일이 일어나는지 알 수 있도록 해줍니다. 하지만 당신이 계속 관여해야 합니다. 올바른 환경 변수나 유효한 데이터 소스를 제공하지 않으면 멈출 수 있습니다. YOLO 에이전트는 추가 단계를 건너뛰고 앞으로 돌진합니다. 이는 빠르게 결과를 원하는 바이브 코딩 프로젝트에 훌륭합니다. 하지만 일부 구조, 테스트, 체크포인트를 희생하게 됩니다.

한 가지 강조하고 싶은 것은, 전략적 에이전트가 완료하지 못했다고 해서 더 나쁘다는 의미는 아니라는 것입니다. 단지 이 특정 시나리오에 적합한 도구가 아니었다는 것입니다. 만약 정확성과 명확성이 속도보다 더 중요한 프로덕션 코드베이스에서 작업하고 있었다면, 저는 절대적으로 전략적 에이전트를 선호했을 것입니다. 어쨌든, 이것이 서로 다른 에이전트 프로필이 Warp에서의 경험을 어떻게 바꿀 수 있는지에 대한 간단한 살펴보기였습니다. 이 영상이 도움이 되었기를 바랍니다.

## 핵심 요약

- **에이전트 프로필**: Warp에서 코딩 에이전트의 동작을 상황에 맞게 설정할 수 있는 기능
- **전략적 에이전트**: 체계적이고 투명하며 설정 가능. 사용자 입력을 요청하고 상세한 계획을 수립하지만 적극적인 관여가 필요
- **YOLO 에이전트**: 가드레일이 적고 빠르게 진행. 권한 요청 없이 핵심 기능에 집중하지만 구조와 검증을 일부 희생
- **트레이드오프**: 프로덕션 환경에서는 전략적 에이전트가, 빠른 프로토타입이 필요한 경우 YOLO 에이전트가 적합
- **실제 사례**: NFL 예측 앱 구축에서 전략적 에이전트는 상세한 계획을 세웠지만 데이터 소스 문제로 멈췄고, YOLO 에이전트는 안정적인 데이터 소스에 집중해 작동하는 결과물을 빠르게 생성
