# Github MCP 서버 사용하기

[영상 바로가기](https://youtu.be/OXYQyNXH2Bw)

## 전체 번역

GitHub MCP 서버를 사용하면 Warp 에이전트가 여러분의 GitHub 저장소에서 직접 읽고 쓸 수 있습니다. 설정하려면 먼저 GitHub 개인 액세스 토큰이 필요합니다.

토큰을 얻는 방법은 다음과 같습니다. GitHub 계정에 들어가서 설정으로 이동한 다음, 개발자 설정으로 들어가서 개인 액세스 토큰을 클릭합니다. 토큰을 생성할 때 repo 아래의 모든 권한을 활성화하고 read user도 활성화하세요. 저는 AI에게 너무 많은 쓰기 권한을 주지 않기 위해 스코프를 최소한으로 유지하는 것을 선호합니다. 그렇게 하면 AI가 실수로 변경 사항을 커밋하거나 중요한 내용을 덮어쓸 위험이 줄어듭니다. 토큰을 얻었으면 Warp로 이동하세요.

명령 팔레트에서 MCP 패널을 열고 추가를 클릭합니다. GitHub MCP 서버의 JSON 설정을 붙여넣으세요. 방금 받은 액세스 토큰을 입력한 다음 저장을 누릅니다. 그러면 사용 가능한 모든 엔드포인트가 표시됩니다. 이제 설정이 완료되었으니 제가 항상 사용하는 두 가지 실제 워크플로우를 보여드리겠습니다. 첫 번째는 모든 열린 PR 요약하기입니다. 여기서 저는 Warp의 내부 GitHub 저장소 안에 있습니다. 에이전트에게 모든 열린 풀 리퀘스트의 논의 내용과 주요 미해결 질문을 요약하도록 요청하는 프롬프트를 실행할 수 있습니다. 내부적으로 MCP 서버는 list pull request를 호출하여 PR을 가져온 다음, get pull request comments와 get pull request reviews를 호출하여 모든 세부 정보를 가져옵니다. 모든 내용을 깔끔한 요약으로 정리해줍니다. 그리고 무엇보다 좋은 점은 각 PR에 대한 클릭 가능한 링크를 제공하여 바로 들어가서 리뷰할 수 있다는 것입니다. 두 번째 워크플로우는 to-do에서 이슈 생성하기입니다. 이 경우 저는 Warp Drive에 저장된 프롬프트를 사용하고 있으며, 이 프롬프트는 에이전트에게 저장소의 모든 to-do 주석을 찾아서 각각에 대한 GitHub 이슈를 생성하도록 지시합니다. Warp 에이전트는 코드베이스를 스캔하여 코드 내의 모든 to-do를 찾은 다음, 각각에 대해 GitHub MCP 서버의 create issue를 자동으로 호출합니다. 완료되면 모든 새 이슈의 클릭 가능한 목록을 얻게 되며, 각 이슈에는 원래 to-do 텍스트와 코드로 돌아가는 링크가 포함됩니다.

이 두 워크플로우는 각각 수동으로 처리하는 것에 비해 약 20~30분을 절약해줍니다.

## 핵심 요약

- GitHub MCP 서버를 사용하면 Warp 에이전트가 GitHub 저장소와 직접 상호작용 가능
- 설정에는 최소 권한의 GitHub 개인 액세스 토큰 필요 (repo + read user)
- 실용적인 워크플로우 1: 모든 열린 PR의 논의 내용과 미해결 질문 자동 요약
- 실용적인 워크플로우 2: 코드의 모든 to-do 주석을 자동으로 GitHub 이슈로 변환
- 각 워크플로우당 20~30분의 시간 절약 효과
