# MCP로 Warp와 Linear 연결하기

[영상 바로가기](https://youtu.be/jxeMfuS1pXk)

## 전체 자막

안녕하세요, 오늘은 Warp에서 MCP를 설정하여 Warp의 에이전트가 외부 데이터 소스에 접근할 수 있도록 하는 방법을 보여드리겠습니다.

MCP는 Anthropic의 모델 컨텍스트 프로토콜(Model Context Protocol)입니다. Warp에서 설정하기 매우 쉽습니다. 먼저 설정되지 않은 상황을 보여드리겠습니다. 예를 들어, 에이전트에게 제게 할당된 모든 Linear 작업을 알려달라고 요청한다고 해봅시다. 지금은 설정이 되어 있지 않기 때문에 질문에 답할 수 없습니다.

Warp에서 MCP를 설정하려면 Warp Drive로 이동합니다. 개인(Personal) 섹션까지 스크롤을 내려서 MCP 서버(MCP servers)를 클릭합니다. 또는 Command+P를 눌러 MCP 서버 팔레트를 여는 방법도 있습니다. 어느 방법이든 작동합니다. 제게는 이미 여러 개가 설정되어 있는 것을 볼 수 있고, 여기서 새로운 것을 추가하려고 합니다.

JSON 파일을 가져오겠습니다. 파일에 저장해 두었습니다. Warp에서 파일을 열려면 Command+O를 누릅니다. 이 Linear MCP JSON 파일을 열겠습니다. JSON은 다음과 같이 생겼습니다. 여기서 "추가(Add)"를 클릭하고 이것을 복사해서 붙여넣겠습니다.

"저장(Save)"을 클릭합니다. 이렇게 하면 서버가 저장되고 동시에 시작됩니다. 이제 Linear MCP 서버가 실행되고 있는 것을 볼 수 있습니다. 이제 창을 닫고 같은 질문을 다시 시도해보겠습니다. 이제 MCP를 사용하여 답변을 할 수 있어야 합니다. 제 얼굴을 치우겠습니다.

보시다시피 MCP 도구를 호출하고 있습니다. 실제 원시 응답을 보고 싶으면 Warp에서는 언제든지 클릭해서 확인할 수 있습니다. 이건 작동하지 않았네요.

저를 찾지 못했습니다. 그럼 다른 방법으로 찾을 수 있는지 볼까요? 오, 팀의 다른 사람을 찾았네요. 괜찮습니다.

왜 저를 못 찾았을까요? 아마 다른 주소로 가입했기 때문일 수도 있습니다.

아, 있습니다. 올바른 사용자를 찾았습니다.

저희 에이전트의 멋진 점 중 하나는 자기 수정(self-correcting)을 매우 잘한다는 것입니다.

여기서 제 이슈들을 모두 가져올 것입니다. 꽤 많을 수도 있습니다.

들여다볼 수 있습니다. 오, 정말 많네요.

좋습니다, 여기 있습니다. 매우 간단합니다. 몇 가지 다른 사항을 언급하자면, "중지(Stop)"를 눌러 MCP 서버를 언제든지 중지할 수 있습니다. "편집(Edit)"을 클릭하여 편집할 수도 있습니다. 서버 시작에 오류가 있는 경우, Warp에서 매우 유용하게 할 수 있는 것은 원시 로그를 볼 수 있다는 것입니다. 이렇게 하면 MCP 서버의 로그를 추적(tail)하는 터미널 세션이 시작됩니다.

일반적으로 이러한 MCP 서버는 중지할 때까지 계속 실행됩니다. Warp를 닫았다가 다시 열어도 실행 상태가 유지됩니다. MCP 설정을 즐겁게 해보세요.

## 핵심 요약

- **MCP 설정**: Warp Drive → Personal → MCP Servers 또는 Command+P로 접근
- **서버 추가**: JSON 설정 파일을 복사하여 붙여넣고 저장하면 자동으로 시작
- **Linear 연동**: MCP를 통해 Warp 에이전트가 Linear 작업을 조회하고 관리 가능
- **자기 수정**: Warp 에이전트는 초기 실패 후 자동으로 다시 시도하여 문제 해결
- **서버 관리**: 중지, 편집, 로그 확인 기능 제공
- **지속성**: MCP 서버는 Warp를 닫아도 실행 상태 유지
